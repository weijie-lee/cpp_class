>
> 上节课我们用DP解决了斐波那契和爬楼梯，今天我们来挑战一道更有意思的经典题——**数字三角形**！
>

### 2. 故事引入——下山捡金币（10 分钟）

> 想象你站在一座金币山的山顶⛰️。山上铺满了金币，排成一个三角形的样子：
>
> ```
>        2
>       3 4
>      6 5 7
>     4 1 8 3
> ```
>
> 你从山顶出发，每次只能往**左下**或**右下**走一步，一直走到山脚。经过的每个位置都能捡起金币。
>
> 问题来了：**怎么走，才能捡到最多的金币呢？**
>
> 这就是大名鼎鼎的"数字三角形"问题！

#### 先想笨办法

> 最笨的方法是什么？——把所有可能的路线都试一遍，比较哪条路金币最多。
>
> 但如果山有20层，路线就有 2^19 = 524288 条！太多了，算到天黑都算不完。
>
> 还记得上节课的经验吗？遇到这种"重复计算"的问题，就该请出我们的好朋友——**动态规划**！

#### 换一个聪明的想法：从山脚开始想

> 我们换个角度：不从山顶出发，而是**从山脚开始，往上想**。
>
> 站在倒数第二行的每个位置，你只需要看脚下的两条路——左下和右下——哪条金币更多，就选哪条。
>
> 就像你在商场选电梯一样：左边的电梯排了3个人，右边排了8个人，你肯定选左边（人少的）。我们这里相反，要**选金币多的**！
>
> 这样一层一层往上推，最后到山顶，就能知道最多能捡多少金币了！

### 3. 知识讲解——用二维数组存三角形（10 分钟）

> 我们先来解决第一个问题：怎么把三角形存到电脑里？
>
> 用**二维数组** `a[i][j]`：
> - `i` 表示第几行（从第1行开始）
> - `j` 表示这一行的第几个数
> - 第 `i` 行恰好有 `i` 个数
>
> ```
> 行号   三角形        数组存储
>  1       2           a[1][1]=2
>  2      3 4          a[2][1]=3  a[2][2]=4
>  3     6 5 7         a[3][1]=6  a[3][2]=5  a[3][3]=7
>  4    4 1 8 3        a[4][1]=4  a[4][2]=1  a[4][3]=8  a[4][4]=3
> ```

#### 自底向上的DP思路

> 再来一个数组 `dp[i][j]`，表示**从第 i 行第 j 个位置出发，走到山脚，最多能捡多少金币**。
>
> **初始值**：最后一行就是山脚，不用再往下走了，所以 `dp[4][j] = a[4][j]`。
>
> **状态转移**（从倒数第二行开始，往上推）：
> ```
> dp[i][j] = max(dp[i+1][j], dp[i+1][j+1]) + a[i][j]
> ```
> 翻译成大白话：**"我这个位置的最大金币 = 下面两条路中金币多的那条 + 我自己位置的金币"**
>
> （在黑板上手动模拟整个过程，让学生跟着一起算）
>
> ```
> 原始三角形：          dp数组（自底向上填）：
>      2                     20
>     3 4                  11  14
>    6 5 7                10  13  15
>   4 1 8 3               4   1   8   3   ← 初始值就是本身
> ```
>
> 所以最多能捡到 **20** 个金币！路线是 2→4→7→8。

### 4. 拓展了解——01背包思想（仅故事介绍，不要求掌握）

## 五、课堂例题

### 例题1：3行小三角形（入门版）

**题目**：给定一个3行的数字三角形，求从顶部到底部的最大路径和。

```
  1
 3 2
4 10 1
```

**讲解思路**：

1. 先把三角形存到数组里：`a[1][1]=1`, `a[2][1]=3, a[2][2]=2`, `a[3][1]=4, a[3][2]=10, a[3][3]=1`
2. 初始值：`dp[3][1]=4, dp[3][2]=10, dp[3][3]=1`
3. 倒推第2行：
   - `dp[2][1] = max(4, 10) + 3 = 13`
   - `dp[2][2] = max(10, 1) + 2 = 12`
4. 倒推第1行：
   - `dp[1][1] = max(13, 12) + 1 = 14`
5. 答案：14，路线 1→3→10

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    // 3行的数字三角形，直接写在代码里
    int n = 3;
    int a[4][4] = {};  // 从下标1开始用，多开一点
    int dp[4][4] = {};

    // 存入三角形的数据
    a[1][1] = 1;
    a[2][1] = 3;  a[2][2] = 2;
    a[3][1] = 4;  a[3][2] = 10; a[3][3] = 1;

    // 第一步：初始值——最后一行直接抄过来
    for (int j = 1; j <= n; j++) {
        dp[n][j] = a[n][j];
    }

    // 第二步：从倒数第二行往上推
    for (int i = n - 1; i >= 1; i--) {
        for (int j = 1; j <= i; j++) {
            dp[i][j] = max(dp[i + 1][j], dp[i + 1][j + 1]) + a[i][j];
        }
    }

    // 第三步：输出山顶的值，就是答案
    cout << dp[1][1] << endl;

    return 0;
}
```

**输出**：`14`

> 💡 **给同学们的话**：这道题只有3行，你甚至可以用手算验证程序的结果对不对。手算和电脑算出一样的答案，说明我们的思路是对的！

### 例题2：n行通用数字三角形（标准版）

**题目**：输入一个 n 行的数字三角形（n ≤ 100），第 i 行有 i 个整数。求从顶部到底部的最大路径和。

**输入样例**：
```
4
2
3 4
6 5 7
4 1 8 3
```

**输出样例**：
```
20
```

**讲解思路**：

和例题1思路完全一样，只不过现在需要用 `cin` 读入数据，行数 n 由用户输入。

**DP四步法分析**：

| 步骤 | 内容 |
|------|------|
| 找状态 | `dp[i][j]` = 从第 i 行第 j 列出发到山脚的最大金币数 |
| 找转移 | `dp[i][j] = max(dp[i+1][j], dp[i+1][j+1]) + a[i][j]` |
| 定初值 | `dp[n][j] = a[n][j]`（最后一行直接就是自己） |
| 写代码 | 两层循环，外层从第 n-1 行到第 1 行，内层从第 1 列到第 i 列 |

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int a[105][105];   // 存三角形的数据
int dp[105][105];  // 存DP结果

int main() {
    int n;
    cin >> n;

    // 读入三角形
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= i; j++) {
            cin >> a[i][j];
        }
    }

    // 初始值：最后一行
    for (int j = 1; j <= n; j++) {
        dp[n][j] = a[n][j];
    }

    // 自底向上推
    for (int i = n - 1; i >= 1; i--) {
        for (int j = 1; j <= i; j++) {
            dp[i][j] = max(dp[i + 1][j], dp[i + 1][j + 1]) + a[i][j];
        }
    }

    // 输出答案
    cout << dp[1][1] << endl;

    return 0;
}
```

**输出**：`20`

> 💡 **代码解读**：
> - 数组开 `105` 是为了保险，比 100 多一点，防止越界。
> - 外层循环 `i` 从 `n-1` 到 `1`，是"自底向上"的体现。
> - 内层循环 `j` 从 `1` 到 `i`，因为第 i 行只有 i 个数。
> - `max()` 函数帮我们挑两条路中金币多的那条——这就是"贪心选择"在DP里的体现。

### 练习题1（基础巩固）⭐

**题目**：手动填写下面这个数字三角形的 dp 数组，并写出最大路径和。

```
    5
   8 3
  12 7 16
 4  10 11 6
```

请在纸上写出 dp 数组的每一个值，最后给出答案。

**参考答案**：

```
dp数组：
     35
   30  30
  16  21  27
   4  10  11   6   ← 初始值

最大路径和：35
路线：5 → 8 → 12 → 10  或  5 → 8 → 7 → 10（都不对，让我们仔细看）
实际路线：5 → 3 → 16 → 11 = 35
```

> 这题的陷阱是：不能每一步都选大的（贪心不一定对），必须用DP全局考虑！这正是DP的魅力所在。

### 练习题3（上机编程）⭐⭐

**题目**：输入一个 n 行的数字三角形（n ≤ 50），求从顶到底的**最小路径和**。

**输入样例**：
```
3
5
9 2
3 8 1
```

**输出样例**：
```
8
```

**提示**：只需要把例题代码中的一个地方改一下，想想是哪里？

**参考答案**：把 `max` 改成 `min`！路线：5→2→1=8。

```cpp
dp[i][j] = min(dp[i + 1][j], dp[i + 1][j + 1]) + a[i][j];
```

### 练习题4（选做挑战）⭐⭐⭐

**题目**：在例题2的基础上，不仅输出最大路径和，还要输出具体的路线（从第1行到第n行，每行选了第几列）。

**输入样例**：
```
4
2
3 4
6 5 7
4 1 8 3
```

**输出样例**：
```
20
1 2 3 3
```

**提示**：额外用一个数组 `path[i][j]` 记录从位置 `(i,j)` 出发时选择了下一行的哪一列。在填 dp 表的时候顺便记录下来。

**参考代码**：

```cpp
#include <iostream>
using namespace std;

int a[105][105], dp[105][105];
int path[105][105];  // 记录选择：往左下还是右下

int main() {
    int n;
    cin >> n;
    for (int i = 1; i <= n; i++)
        for (int j = 1; j <= i; j++)
            cin >> a[i][j];

    for (int j = 1; j <= n; j++)
        dp[n][j] = a[n][j];

    for (int i = n - 1; i >= 1; i--) {
        for (int j = 1; j <= i; j++) {
            if (dp[i + 1][j] >= dp[i + 1][j + 1]) {
                dp[i][j] = dp[i + 1][j] + a[i][j];
                path[i][j] = j;       // 选左下
            } else {
                dp[i][j] = dp[i + 1][j + 1] + a[i][j];
                path[i][j] = j + 1;   // 选右下
            }
        }
    }

    cout << dp[1][1] << endl;

    // 输出路线
    int col = 1;
    for (int i = 1; i <= n; i++) {
        if (i > 1) cout << " ";
        cout << col;
        col = path[i][col];  // 跳到下一行选择的列
    }
    cout << endl;

    return 0;
}
```

## 七、常见错误与应对

### 错误1：数组下标从0开始导致混乱

```cpp
// ❌ 错误：用a[0][0]存山顶，下标容易搞混
a[0][0] = 2;

// ✅ 正确：从下标1开始，a[1][1]是山顶，和题目描述一致
a[1][1] = 2;
```

**应对**：告诉学生这类题统一从下标1开始，数组多开几个，宁可浪费一点空间也不要搞混。

---

### 错误2：循环方向搞反

```cpp
// ❌ 错误：自底向上应该i从大到小，写成从小到大了
for (int i = 1; i <= n - 1; i++)

// ✅ 正确：从倒数第二行（n-1）开始，往上推到第1行
for (int i = n - 1; i >= 1; i--)
```

**应对**：提醒学生画图——我们是从山脚往山顶推，所以行号从大到小。

---

### 错误3：内层循环范围错误

```cpp
// ❌ 错误：第i行有i个数，不是n个
for (int j = 1; j <= n; j++)

// ✅ 正确：
for (int j = 1; j <= i; j++)
```

**应对**：让学生回忆三角形的形状——第1行1个数，第2行2个数……第i行i个数。

---

### 错误4：转移方程写反方向

```cpp
// ❌ 错误（这是自顶向下的方程，和我们的循环方向不匹配）：
dp[i][j] = max(dp[i - 1][j - 1], dp[i - 1][j]) + a[i][j];

// ✅ 正确（自底向上）：
dp[i][j] = max(dp[i + 1][j], dp[i + 1][j + 1]) + a[i][j];
```

**应对**：统一教自底向上，让学生记住口诀——**"我站这里，看脚下两条路，选大的加自己"**。

---

### 错误5：忘记初始化最后一行

```cpp
// ❌ 错误：直接开始转移，没有给最后一行赋初值
// dp数组全是0，结果当然不对

// ✅ 正确：先把最后一行的dp值设好
for (int j = 1; j <= n; j++)
    dp[n][j] = a[n][j];
```

**应对**：类比"爬楼梯的第1阶和第2阶"——DP必须有起点（初始值），不然推不动。
