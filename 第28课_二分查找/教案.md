## 二、教学重点与难点

### 重点

- 理解"每次砍掉一半"的核心思想。
- 掌握 `left`、`right`、`mid` 三个变量的含义与更新规则。
- 掌握 `while(left <= right)` 循环条件的含义。

### 难点

- **边界更新**：什么时候 `left = mid + 1`，什么时候 `right = mid - 1`，容易混淆或漏写 `+1` / `-1`。
- **循环条件**：为什么是 `<=` 而不是 `<`，学生容易写错导致漏查或死循环。
- **有序前提**：学生容易忘记先排序就直接二分，导致结果错误。

## 四、讲稿提纲

### 4.1 热身导入——猜数字游戏（5 分钟）

> 同学们，我们来玩一个游戏！老师心里想了一个 1 到 100 之间的数字，你们来猜。每次猜完，老师只会告诉你"大了"或者"小了"。谁能用最少的次数猜到？

- 请一位同学上台来猜。（通常学生会随机猜，需要很多次）
- 再请一位同学，提示他："你能不能每次都猜**中间**的那个数？"
- 引导发现：每次猜中间的数，不管大了还是小了，都能**砍掉一半**的可能！
- **结论**：1 到 100 的数字，最多只要猜 **7 次**就能猜到！（因为 2⁷ = 128 > 100）

> 这种"每次砍掉一半"的方法，有一个很厉害的名字，叫做——**二分查找**！

### 4.2 顺序查找 vs 二分查找（15 分钟）

#### 4.2.1 顺序查找——翻字典的"笨办法"

> 想象一下，你要在一本 1000 页的字典里找一个字。如果你从第 1 页开始，一页一页往后翻，最倒霉的情况要翻多少页？——**1000 页**！这就叫**顺序查找**，一个一个找。

用代码回顾：

```cpp
// 顺序查找：一个一个找
int find(int a[], int n, int x) {
    for (int i = 0; i < n; i++) {
        if (a[i] == x) return i;  // 找到了，返回下标
    }
    return -1;  // 没找到
}
```

> 顺序查找就像排队买奶茶，前面有 100 个人，你就得等 100 轮。**慢！**

#### 4.2.2 二分查找——翻字典的"聪明办法"

> 但是！字典里的字是**按拼音顺序排好的**。所以你可以先翻到中间，看看要找的字在前半本还是后半本，然后再翻剩下那半本的中间……每次都砍掉一半！1000 页的字典，最多翻 **10 次**就能找到！（因为 2¹⁰ = 1024 > 1000）

**效率对比表**（板书 / PPT）：

| 数据量 | 顺序查找（最多次数） | 二分查找（最多次数） |
|--------|----------------------|----------------------|
| 10 | 10 | 4 |
| 100 | 100 | 7 |
| 1,000 | 1,000 | 10 |
| 1,000,000 | 1,000,000 | 20 |

> 一百万个数，顺序查找要找一百万次，二分查找只要 **20 次**！是不是超级厉害？

#### 4.2.3 二分查找的"规矩"——必须有序

> 但是二分查找有一个**非常重要的前提**：数组必须是**排好序的**！
>
> 为什么呢？想想看，如果字典里的字是乱排的，你翻到中间一页，看到的字比你要找的字大，你能确定要找的字在前半本吗？——不能！因为后半本也可能有更小的字。所以如果是乱的，二分查找就**失灵**了。
>
> 记住这句口诀：**"二分查找，有序才行！"**

### 4.3 二分查找代码讲解（10 分钟）

> 现在我们来看看二分查找的代码长什么样。别怕，只有几行！

**关键概念（板书画图）**：

```
数组：[2, 5, 8, 12, 16, 23, 38, 42, 56, 72]
      ↑                    ↑                 ↑
     left                 mid              right

要找的数 x = 23
```

> 想象你面前有一排排好序的数字卡片：
> - `left`（左手）指着最左边
> - `right`（右手）指着最右边
> - `mid`（眼睛看的位置）在正中间
>
> 你看了一眼 mid 位置的数：
> - 如果**刚好等于** x → 太棒了，找到了！
> - 如果 mid 位置的数**比 x 小** → 说明 x 在**右半边**，左手往右移（`left = mid + 1`）
> - 如果 mid 位置的数**比 x 大** → 说明 x 在**左半边**，右手往左移（`right = mid - 1`）
>
> 每次移完手，范围就少了一半！直到找到，或者左手跑到右手的右边去了（说明没找到）。

### 4.4 互动游戏——二分猜数（10 分钟）

**玩法一**：老师想数（写在纸上不给学生看），全班一起用二分法猜。

- 范围 1-100，老师只回答"大了""小了""对了"。
- 要求学生每次都猜 `(left + right) / 2`。
- 板书记录每次的 left、right、mid，让学生直观看到范围在缩小。

**玩法二**：请一位同学想数，老师来猜（故意展示二分过程），其余同学验证老师有没有算错 mid。

> 你们看，老师最多 7 次就猜到了！以后你们和朋友玩猜数字游戏，用这招，赢定了！

## 五、课堂例题

### 例题 1：基础二分查找（查找目标值）

**题目**：给定一个从小到大排好序的数组，和一个要查找的数 x。如果 x 在数组中，输出它的下标（从 0 开始）；如果不在，输出 -1。

**输入样例**：
```
10 23
2 5 8 12 16 23 38 42 56 72
```

**输出样例**：
```
5
```

**讲解思路**：

1. 先读入数组（已经有序，不需要排序）。
2. 设定 `left = 0`，`right = n - 1`。
3. 循环：只要 `left <= right`，就继续找。
4. 算中间位置 `mid = (left + right) / 2`。
5. 比较 `a[mid]` 和 `x`：相等就输出 mid；小了就往右找；大了就往左找。
6. 循环结束还没找到，输出 -1。

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, x;
    cin >> n >> x;

    int a[100005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    // 二分查找
    int left = 0, right = n - 1;
    int ans = -1;  // 默认没找到

    while (left <= right) {
        int mid = (left + right) / 2;

        if (a[mid] == x) {
            ans = mid;   // 找到了！
            break;       // 跳出循环
        } else if (a[mid] < x) {
            left = mid + 1;   // x 在右半边，左手右移
        } else {
            right = mid - 1;  // x 在左半边，右手左移
        }
    }

    cout << ans << endl;
    return 0;
}
```

**逐行带读要点**：

- `left = 0, right = n - 1`：一开始整个数组都是查找范围。
- `while (left <= right)`：只要左手没有跑到右手右边，就还有数没看过。
- `mid = (left + right) / 2`：取中间位置（整数除法，自动向下取整）。
- `left = mid + 1`：为什么要 `+1`？因为 `mid` 这个位置已经看过了，不用再看了！同理 `right = mid - 1`。

### 例题 2：二分查找——找不到的情况 & 统计次数

**题目**：给定一个有序数组和 q 个查询，对于每个查询的数 x，输出它在数组中的下标（从 1 开始），如果不存在输出 `0`。同时统计每次查找比较了多少次。

**输入样例**：
```
8 3
1 3 5 7 9 11 13 15
5 10 13
```

**输出样例**：
```
3 3
0 4
7 3
```

**讲解思路**：

1. 这题有多个查询，用一个循环处理每次查询。
2. 每次查找前把计数器清零，每比较一次就加 1。
3. 注意题目下标从 1 开始，所以找到后输出 `mid + 1`。
4. 通过输出比较次数，让学生直观感受二分查找的效率。

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, q;
    cin >> n >> q;

    int a[100005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    while (q--) {
        int x;
        cin >> x;

        int left = 0, right = n - 1;
        int ans = 0;   // 0 表示没找到（题目下标从1开始）
        int cnt = 0;   // 比较次数计数器

        while (left <= right) {
            int mid = (left + right) / 2;
            cnt++;  // 每次比较，计数器+1

            if (a[mid] == x) {
                ans = mid + 1;  // 下标从1开始
                break;
            } else if (a[mid] < x) {
                left = mid + 1;
            } else {
                right = mid - 1;
            }
        }

        cout << ans << " " << cnt << endl;
    }

    return 0;
}
```

**课堂互动**：

> 同学们看，数组有 8 个数，我们最多比较了几次？——**4 次！** 如果用顺序查找呢？最多要 **8 次**。数据越多，差距越大！

### 例题 3（进阶）：在有序数组中找第一个 ≥ x 的位置

**题目**：给定一个从小到大排好序的数组（可能有重复元素），找出第一个大于等于 x 的元素的下标（从 1 开始）。如果所有元素都小于 x，输出 `n + 1`。

**输入样例**：
```
8 5
1 2 4 4 5 5 7 9
```

**输出样例**：
```
5
```

**讲解思路**：

> 这道题和前面不太一样。前面是找"等于 x 的数"，这道题是找"第一个大于等于 x 的数"。
>
> 比喻：你去超市买苹果，预算是 5 块钱。货架上的苹果按价格从低到高排好了：1元、2元、4元、4元、5元、5元、7元、9元。你想找到**第一个你买得起的苹果**（价格 ≥ 5 的第一个）。

**关键变化**：

- 当 `a[mid] >= x` 时，mid 可能是答案，但左边可能还有更靠前的，所以**记下来但继续往左找**（`right = mid - 1`）。
- 当 `a[mid] < x` 时，mid 肯定不是答案，往右找（`left = mid + 1`）。

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, x;
    cin >> n >> x;

    int a[100005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    int left = 0, right = n - 1;
    int ans = n + 1;  // 默认：所有数都比 x 小

    while (left <= right) {
        int mid = (left + right) / 2;

        if (a[mid] >= x) {
            ans = mid + 1;      // 记下这个位置（下标从1开始）
            right = mid - 1;    // 但继续往左找，看有没有更靠前的
        } else {
            left = mid + 1;     // 太小了，往右找
        }
    }

    cout << ans << endl;
    return 0;
}
```

**对比强调**：

| | 例题 1（找等于 x） | 例题 3（找第一个 ≥ x） |
|---|---|---|
| 找到相等时 | 直接 `break` | **不 break**，记下答案继续往左找 |
| 初始 ans | -1 | n + 1 |
| 适用场景 | 查一个数在不在 | 查位置 / 有重复元素 |

> 同学们，例题 3 是二分查找的"升级版"，竞赛里特别常用。今天能听懂就很棒了，不用着急一下子全掌握！

## 七、常见错误与应对

### 错误 1：循环条件写成 `left < right`（漏了等号）

```cpp
// ❌ 错误写法
while (left < right) { ... }

// ✅ 正确写法
while (left <= right) { ... }
```

**后果**：当 `left == right` 时，还有一个数没有检查就退出了，导致漏查。

**课堂应对**：让学生在纸上模拟一个只有 1 个元素的数组查找过程，发现用 `<` 会直接跳过。

> 记忆口诀：**"左小于等于右，一个都不能漏！"**

---

### 错误 2：边界更新忘记 +1 / -1

```cpp
// ❌ 错误写法
left = mid;      // 没有 +1
right = mid;     // 没有 -1

// ✅ 正确写法
left = mid + 1;
right = mid - 1;
```

**后果**：`mid` 已经比较过了还留在范围里，可能导致**死循环**（left 和 right 不再变化）。

**课堂应对**：故意演示一次死循环，让学生看到程序"卡住了"，印象深刻。

> 比喻：你翻字典看了第 50 页，发现不是，下次还从第 50 页开始翻？那你永远也翻不完！

---

### 错误 3：对无序数组使用二分查找

```cpp
// ❌ 数组没有排序就二分
int a[] = {5, 2, 8, 1, 9, 3};
// 直接二分查找 → 结果不可靠！
```

**课堂应对**：先给学生一个无序数组，让他们尝试二分查找一个存在的数，发现找不到。

> **口诀**：**"二分之前先排序，无序二分白忙活！"**

---

### 错误 4：mid 计算溢出（了解即可）

```cpp
// 当 left 和 right 都很大时，left + right 可能超出 int 范围
// 更安全的写法：
int mid = left + (right - left) / 2;
```

**课堂处理**：简单提一句即可，告诉学生 CSP-J 的数据范围一般不会触发这个问题，但知道有这回事。

---

### 错误 5：找到后忘记 break（基础版）

```cpp
if (a[mid] == x) {
    ans = mid;
    // 忘记写 break; → 程序还会继续循环，虽然结果可能对，但浪费时间
}
```

**课堂应对**：提醒学生基础版找到就可以停了，养成好习惯。
