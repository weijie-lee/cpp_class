## 二、教学重点与难点

**重点：**
- 模拟法的核心思想：题目说什么，代码就做什么——"照着剧本演"。
- 把文字描述翻译成代码步骤的能力（读题 → 拆步骤 → 写代码）。
- 用变量跟踪状态变化，理解"程序就是在不断改变变量的值"。

**难点：**
- 从自然语言到代码的"翻译"过程，学生容易跳步或遗漏边界条件。
- 进位/借位等需要多个变量联动变化的场景（如时钟 59 分 → 60 分要归零并且小时 +1）。
- 循环中什么时候该更新变量、更新的先后顺序。

## 四、讲稿提纲

### 4.1 复习引入（5 分钟）

> "同学们，上两节课我们学了枚举法——把所有可能都试一遍，像一个个抽屉都拉开看看。今天我们来学一个新方法，叫做**模拟法**。"

提问回顾：
- "枚举法是怎么解题的？"（引导回忆：穷举所有情况）
- "如果题目不是让你找答案，而是让你**按照规则做一件事**呢？"

### 4.2 讲解：什么是模拟法（15 分钟）

**核心比喻——当导演拍电影：**

> "想象你是一个导演🎬，拿到了一个剧本。剧本上写着：
> - 第一步：演员走到门口
> - 第二步：演员敲三下门
> - 第三步：门打开了，演员走进去
>
> 你要做的就是**让演员严格按照剧本，一步一步演出来**。
>
> 模拟法就是这样！题目就是剧本，你的代码就是演员。题目说先做什么、再做什么，你就老老实实写代码让电脑一步一步照着做。"

**和枚举法的区别：**

| | 枚举法 | 模拟法 |
|---|---|---|
| 干什么 | 把所有情况试一遍 | 按题目描述一步步做 |
| 像什么 | 开所有抽屉找东西 | 当导演按剧本拍戏 |
| 关键 | 不遗漏 | 不跳步 |

**翻译三步法（板书）：**

> "做模拟题，我们有一个秘诀，叫'翻译三步法'："
>
> 1. **读剧本** —— 仔细读题，把每一步用自己的话说出来。
> 2. **列步骤** —— 把每一步写成"先…再…然后…"，像写菜谱一样。
> 3. **写代码** —— 每一步翻译成一行或几行 C++ 代码。

**生活案例——泡方便面：**

> "题目：把方便面泡好。
> - 第一步：撕开包装
> - 第二步：把面饼放进碗里
> - 第三步：倒入调料包
> - 第四步：倒入开水
> - 第五步：盖上盖子等 3 分钟
>
> 这就是模拟！每一步都不能跳，顺序也不能乱。你不能先倒开水再撕包装对吧？"

**变量跟踪——用"记分板"来理解：**

> "模拟法经常需要你用变量来记住'现在是什么状态'。就像篮球比赛的记分板，每次进球分数就变一下。我们写代码的时候也一样，用变量记住'现在的值是多少'，每一步可能都要更新它。"

介绍**变量跟踪表**（手动模拟）：在纸上画表格，每一步写下每个变量的当前值，像这样：

| 步骤 | n | result |
|------|---|--------|
| 初始 | 123 | 0 |
| 第1步 | 12 | 3 |
| 第2步 | 1 | 32 |
| 第3步 | 0 | 321 |

> "以后我们做模拟题，先在纸上画这个表，把每一步变量的变化写出来，代码就好写多了！"

### 4.3 例题讲解穿插（详见第五部分）

### 4.4 总结收尾（5 分钟）

> "今天我们学到了：
> 1. 模拟法 = 当导演，按剧本一步步拍戏
> 2. 翻译三步法：读剧本 → 列步骤 → 写代码
> 3. 用变量跟踪表检查自己的代码对不对
>
> 记住，模拟题最重要的不是聪明，而是**仔细**和**耐心**！"

## 五、课堂例题

### 例题 1：翻转数字（入门级）

**题目：**

输入一个正整数 n（1 ≤ n ≤ 99999），输出它翻转后的结果。例如输入 `1230`，输出 `321`（去掉前导零）。

**讲解思路——翻译三步法演示：**

> "读剧本：把一个数字倒过来。  
> 列步骤：怎么把 `123` 倒过来呢？想象你有一堆积木，每次从右边拿一个放到新的一排——
> 1. 从 n 的末尾取出一位数字（用 `n % 10`）
> 2. 把这个数字放到结果的末尾（`result = result * 10 + 这个数字`）
> 3. 把 n 的末尾砍掉（`n = n / 10`）
> 4. 重复，直到 n 变成 0"

**变量跟踪表（板书演示 n = 1230）：**

| 步骤 | n | digit | result |
|------|---|-------|--------|
| 初始 | 1230 | — | 0 |
| 第1轮 | 123 | 0 | 0 |
| 第2轮 | 12 | 3 | 3 |
| 第3轮 | 1 | 2 | 32 |
| 第4轮 | 0 | 1 | 321 |

> "看！前导零自动就没了，因为 0×10+3 就是 3，不会有前面的 0。"

**完整代码：**

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    
    int result = 0;       // 记分板：翻转后的结果
    while (n > 0) {       // 只要剧本没演完（n还有数字）
        int digit = n % 10;           // 第一步：取出末尾数字
        result = result * 10 + digit;  // 第二步：放到结果末尾
        n = n / 10;                    // 第三步：砍掉末尾
    }
    
    cout << result << endl;
    return 0;
}
```

**师生互动点：**
- 请一个同学上来，在纸上当"人肉电脑"，手动跟踪 n = 520 的执行过程。
- 提问："如果输入是 100，结果是多少？"（答：1。提醒学生注意前导零的处理是自动的。）

### 例题 2：时钟进位（进阶级）

**题目：**

一个电子时钟显示 HH:MM:SS 格式的时间（用 24 小时制）。给定当前时间的时 h、分 m、秒 s，再给定经过的秒数 t（0 ≤ t ≤ 100000），请输出 t 秒后时钟显示的时间（格式 HH:MM:SS，不足两位补前导零）。

**输入示例：** `23 59 45 20`  
**输出示例：** `00:00:05`

**讲解思路：**

> "想象你家墙上的时钟⏰，秒针一格一格走。每走一格：
> - 秒数 +1
> - 如果秒到了 60？归零！分钟 +1
> - 如果分钟到了 60？归零！小时 +1
> - 如果小时到了 24？归零！（新的一天）
>
> 这就是进位！和我们十进制加法的进位是一样的道理。"

**方法一：直接一秒一秒模拟（最直观，但 t 大时慢）**

> "最笨但最好理解的办法：每次让秒 +1，然后检查要不要进位。循环 t 次。"

**方法二：数学转换（推荐，更快）**

> "聪明的办法：把时间全部变成秒数，加上 t，再变回时分秒。就像把人民币全换成'分'来算，最后再换回'元角分'。"

**完整代码（方法二）：**

```cpp
#include <iostream>
#include <iomanip>   // 用来补前导零
using namespace std;

int main() {
    int h, m, s, t;
    cin >> h >> m >> s >> t;
    
    // 第一步：把现在的时间全部变成秒数（统一单位）
    int total = h * 3600 + m * 60 + s;
    
    // 第二步：加上经过的秒数
    total = total + t;
    
    // 第三步：处理超过一天的情况（24小时 = 86400秒）
    total = total % 86400;
    
    // 第四步：把总秒数变回时、分、秒
    h = total / 3600;           // 有多少个小时
    m = (total % 3600) / 60;    // 剩下的有多少个分钟
    s = total % 60;             // 最后剩下多少秒
    
    // 第五步：按格式输出（不足两位补0）
    cout << setfill('0') << setw(2) << h << ":"
         << setw(2) << m << ":"
         << setw(2) << s << endl;
    
    return 0;
}
```

**补充讲解 `setfill` 和 `setw`：**

> "`setw(2)` 是说'这个数字至少占 2 格'，`setfill('0')` 是说'如果不够就用 0 来填'。就像学校运动会号码牌，01 号、02 号，前面要补零。"

**师生互动点：**
- 在黑板上手动演算 23:59:45 + 20秒 的过程。
- 提问："如果 total 超过了 86400（一天的秒数），为什么要取余？"（答：时钟不会显示 25 点，到 24 就归零重新来。）

### 例题 3：简单日期推算（挑战级）

**题目：**

输入年份 y 和一个天数 d（1 ≤ d ≤ 365），输出这一年的第 d 天是几月几日。不考虑闰年，即假设每年都是 365 天，2 月都是 28 天。

**输入示例：** `2024 60`  
**输出示例：** `3 1`（即 3 月 1 日）

**讲解思路：**

> "想象你在撕日历📅。1 月有 31 页，你一页页撕。如果你要撕到第 60 页：
> - 先撕 1 月的 31 页，还剩 60-31=29 页要撕
> - 再撕 2 月的 28 页，还剩 29-28=1 页
> - 到 3 月了，只需要撕 1 页 → 答案是 3 月 1 日！
>
> 这个'一个月一个月减'的过程，就是模拟！"

**列步骤：**
1. 用一个数组存每个月有多少天。
2. 从 1 月开始，看 d 够不够减去这个月的天数。
3. 如果够，就减掉，进入下个月。
4. 如果不够，说明就在这个月，d 就是几号。

**完整代码：**

```cpp
#include <iostream>
using namespace std;

int main() {
    int y, d;
    cin >> y >> d;
    
    // 每个月的天数（不考虑闰年）
    // 下标0不用，1~12分别代表1月~12月
    int days[] = {0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31};
    
    int month = 1;  // 从1月开始撕日历
    
    while (d > days[month]) {   // 如果剩余天数比这个月的总天数多
        d = d - days[month];    // 撕掉这个月，剩余天数减少
        month++;                // 进入下个月
    }
    
    // 循环结束时，d就是当月的第几天
    cout << month << " " << d << endl;
    
    return 0;
}
```

**变量跟踪表（d = 60）：**

| 步骤 | month | d | days[month] | d > days[month]? |
|------|-------|---|-------------|-------------------|
| 初始 | 1 | 60 | 31 | 60 > 31 ✓ |
| 第1轮后 | 2 | 29 | 28 | 29 > 28 ✓ |
| 第2轮后 | 3 | 1 | 31 | 1 > 31 ✗ → 停！ |

> "答案就是 3 月 1 日！是不是和我们撕日历的过程一模一样？"

**师生互动点：**
- 让学生用变量跟踪表手动模拟 d = 32（答案 2 月 1 日）和 d = 365（答案 12 月 31 日）。
- 提问："如果 d = 31，答案是几月几号？"（答：1 月 31 日。因为 31 不大于 31，循环不会执行。）

## 七、常见错误与应对

### 错误 1：循环条件写反或漏掉边界

**典型表现：** 翻转数字时写成 `while (n >= 0)` 导致死循环。

**应对：**
> "n 什么时候应该停？是 n 等于 0 的时候。`n >= 0` 的话，n 等于 0 时还会继续执行，就永远停不下来了。所以应该写 `n > 0`。"

**练习方式：** 让学生在纸上画变量跟踪表，手动走到最后一步，看循环是否正确结束。

---

### 错误 2：变量更新顺序搞混

**典型表现：** 时钟进位先更新了分钟再检查秒数是否溢出，或者先做 `n = n / 10` 再取 `n % 10`。

**应对：**
> "拍电影的时候，演员必须按剧本的顺序来。代码也一样，先做第一步再做第二步，顺序不能反！翻转数字必须**先取出末尾**（% 10），**再砍掉末尾**（/ 10）。你想想，如果先把末尾砍了，你还怎么取呢？"

**练习方式：** 故意展示顺序搞混的代码，让学生找 bug。

---

### 错误 3：忘记初始化变量

**典型表现：** `result` 忘记赋初值 0，导致结果是一个随机数。

**应对：**
> "记分板在比赛开始前一定要归零！如果你不给 `result = 0`，电脑会随便给一个垃圾数字，就像记分板上一场比赛的分数没有清掉一样。"

---

### 错误 4：格式输出不正确

**典型表现：** 时钟输出忘记补前导零，输出 `0:0:5` 而不是 `00:00:05`。

**应对：** 
> "我们可以用 `setw` 和 `setfill` 来补零。如果记不住也没关系，还有一个笨办法：先判断这个数是不是小于 10，如果是，就先输出一个 `0`。"

```cpp
// 笨办法也完全可以：
if (h < 10) cout << "0";
cout << h << ":";
if (m < 10) cout << "0";
cout << m << ":";
if (s < 10) cout << "0";
cout << s << endl;
```

---

### 错误 5：日期推算中的"差一"错误

**典型表现：** 第 31 天输出 2 月 0 日，而不是 1 月 31 日。

**应对：**
> "循环条件是 `d > days[month]` 而不是 `d >= days[month]`。如果剩下的天数**刚好等于**这个月的天数，说明就是这个月的最后一天，不需要再减了。好比你撕日历，1 月有 31 天，第 31 天刚好是 1 月 31 日——你不需要翻到 2 月去！"
