## 二、教学重点与难点

### 重点

- 栈"后进先出"（LIFO）与队列"先进先出"（FIFO）的核心概念区分。
- `stack` 和 `queue` 的头文件引入与五/六个基本操作函数的使用。
- 在操作前判断是否为空（`empty()`），避免对空栈/空队列调用 `top()`/`front()`/`pop()`。

### 难点

- **栈与队列的"取"和"删"是分开的**：`top()`/`front()` 只看不删，`pop()` 只删不看，需要组合使用。初学者容易混淆。
- **括号匹配的逻辑**：遇到左括号入栈、遇到右括号出栈检查——需要将生活直觉转化为代码逻辑。
- **区分栈和队列的 `pop()` 删除位置不同**：栈删栈顶（最后进的），队列删队首（最先进的）。

### 4.1 导入故事（0′–5′）

> 同学们好！今天我们来认识两个新朋友——**栈**和**队列**。
>
> 先讲个小故事：
>
> 🍽️ **场景一**：食堂阿姨洗好了盘子，一个一个往上摞。你去拿盘子的时候，是不是只能拿最上面那个？最后放上去的盘子，反而最先被拿走——这就叫**"后进先出"**，我们给它起个名字叫**栈（zhàn）**。你也可以想象成手枪的弹夹：最后压进去的子弹，最先射出来。
>
> 🧑‍🤝‍🧑 **场景二**：中午去食堂排队打饭，谁先排的队？当然是队伍最前面的人先打到饭。先来的先走——这就叫**"先进先出"**，我们给它起个名字叫**队列**。
>
> 今天我们就来学习怎么用 C++ 来"摞盘子"和"排队"！

### 4.2 讲解一：栈 `stack`（5′–20′）

**要点提纲**：

1. **什么是栈？**
   - 画图演示：竖着的一摞积木，只能从最顶上放、也只能从最顶上拿。
   - 关键词：**后进先出（LIFO — Last In, First Out）**。

2. **C++ 怎么用栈？**
   - 头文件：`#include <stack>`
   - 声明：`stack<int> s;`（尖括号里写放什么类型的数据）
   - 类比：`stack<int>` 就像"一摞写着数字的盘子"。

3. **五个基本操作**（边讲边在黑板画盘子动画）：

   | 操作 | 含义 | 生活比喻 |
   |------|------|---------|
   | `s.push(x)` | 把 x 放到栈顶 | 把一个盘子摞上去 |
   | `s.top()` | 看一眼栈顶是什么（不拿走） | 看看最上面的盘子是哪个 |
   | `s.pop()` | 把栈顶拿走（不告诉你是什么） | 把最上面的盘子端走 |
   | `s.empty()` | 栈里还有没有东西？ | 盘子摞完了吗？ |
   | `s.size()` | 栈里有几个元素 | 数一数还剩几个盘子 |

4. **重要提醒**（红笔标注）：
   - 📌 `top()` 只是"看"，`pop()` 只是"删"，想先看再删要写两行！
   - 📌 对空栈调用 `top()` 或 `pop()` 会出错，操作前先用 `empty()` 检查！

### 4.4 讲解二：队列 `queue`（35′–50′）

**要点提纲**：

1. **什么是队列？**
   - 画图演示：一排小朋友排队，从队尾进、从队首出。
   - 关键词：**先进先出（FIFO — First In, First Out）**。

2. **C++ 怎么用队列？**
   - 头文件：`#include <queue>`
   - 声明：`queue<int> q;`
   - 类比：`queue<int>` 就像"排队买冰淇淋的队伍，每个人身上贴了一个数字"。

3. **六个基本操作**（边讲边画队伍动画）：

   | 操作 | 含义 | 生活比喻 |
   |------|------|---------|
   | `q.push(x)` | 从队尾加入 x | 新同学排到队尾 |
   | `q.front()` | 看一眼队首是谁（不走） | 看看谁排在最前面 |
   | `q.back()` | 看一眼队尾是谁 | 看看谁排在最后面 |
   | `q.pop()` | 队首那个人离开 | 最前面的同学打完饭走了 |
   | `q.empty()` | 队伍里还有人吗？ | 队伍排空了吗？ |
   | `q.size()` | 队伍里有几个人 | 数一数队伍还有多长 |

4. **栈 vs 队列 对比**：
   - 栈——`top()` 看栈顶；队列——`front()` 看队首，`back()` 看队尾。
   - 栈——从顶部进出；队列——从尾部进、从头部出。
   - 都用 `push()` 添加、`pop()` 删除，但删除的位置不同！

### 4.6 讲解三：栈的应用——括号匹配（65′–80′）

**要点提纲**：

1. **生活引入**：

2. **思路讲解**（配合动画演示）：
   - 从左到右一个一个看字符。
   - 遇到 `(`，压入栈——相当于"记住这里有个等待配对的左括号"。

     - 有 → 配对成功，弹出栈顶。
     - 没有（栈为空） → 多了右括号，不匹配！
   - 全部看完后，如果栈为空 → 全部配对成功；栈不为空 → 有多余的左括号。

> | | 栈 Stack | 队列 Queue |
> |---|---|---|

### 例题 1：认识栈的基本操作（入门）

**题目**：依次往栈中放入 1、2、3、4、5，然后一个一个取出来并打印，观察输出顺序。

**讲解思路**：

> 我们往盘子堆上依次放 1 号、2 号、3 号、4 号、5 号盘子。现在从顶上一个个拿，最先拿到的是 5 号——因为它是最后放上去的！

```cpp
#include <iostream>
#include <stack>
using namespace std;

int main() {
    stack<int> s;

    // 依次放入 1, 2, 3, 4, 5
    for (int i = 1; i <= 5; i++) {
        s.push(i);
        cout << "放入: " << i << endl;
    }

    cout << "栈里一共有 " << s.size() << " 个元素" << endl;
    cout << "---开始取出---" << endl;

    // 一个一个取出来
    while (!s.empty()) {
        cout << "取出: " << s.top() << endl;  // 先看
        s.pop();                                // 再删
    }

    cout << "栈空了吗？" << (s.empty() ? "是" : "否") << endl;

    return 0;
}
```

**预期输出**：

```
放入: 1
放入: 2
放入: 3
放入: 4
放入: 5
栈里一共有 5 个元素
---开始取出---
取出: 5
取出: 4
取出: 3
取出: 2
取出: 1
栈空了吗？是
```

**要点强调**：放入顺序 1→5，取出顺序 5→1，完美体现"后进先出"！

### 例题 2：认识队列的基本操作（入门）

**题目**：模拟 5 位同学排队：依次编号 1~5 进入队列，然后一个一个离开，观察输出顺序。

**讲解思路**：

> 食堂开门了！1 号同学先到，排在最前面；2 号第二个到……5 号最后到。现在开始打饭，谁先打？当然是 1 号！

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    queue<int> q;

    // 5 位同学依次排队
    for (int i = 1; i <= 5; i++) {
        q.push(i);
        cout << i << " 号同学排到队尾" << endl;
    }

    cout << "队伍长度: " << q.size() << endl;
    cout << "队首: " << q.front() << " 号" << endl;
    cout << "队尾: " << q.back() << " 号" << endl;
    cout << "---开始打饭---" << endl;

    // 一个一个离开
    while (!q.empty()) {
        cout << q.front() << " 号同学打完饭走了" << endl;
        q.pop();
    }

    cout << "队伍空了吗？" << (q.empty() ? "是" : "否") << endl;

    return 0;
}
```

**预期输出**：

```
1 号同学排到队尾
2 号同学排到队尾
3 号同学排到队尾
4 号同学排到队尾
5 号同学排到队尾
队伍长度: 5
队首: 1 号
队尾: 5 号
---开始打饭---
1 号同学打完饭走了
2 号同学打完饭走了
3 号同学打完饭走了
4 号同学打完饭走了
5 号同学打完饭走了
队伍空了吗？是
```

**要点强调**：进入顺序 1→5，离开顺序也是 1→5，完美体现"先进先出"！与栈正好相反。

### 例题 3：括号匹配（栈的应用，进阶）

**题目**：输入一个只包含 `(` 和 `)` 的字符串，判断括号是否匹配。

**讲解思路**：

> 想象你有一个盒子（栈）。从左到右读字符串——
> - 看到 `(`：放一个小球进盒子，代表"这里有个左括号在等配对"。
> - 看到 `)`：去盒子里拿一个小球出来配对。如果盒子是空的，说明右括号多了！
> - 读完后看盒子：空的说明全部配对完毕，还有小球说明左括号多了。

```cpp
#include <iostream>
#include <stack>
#include <string>
using namespace std;

int main() {
    string str;
    cout << "请输入括号字符串: ";
    cin >> str;

    stack<char> s;
    bool ok = true;  // 记录是否匹配

    for (int i = 0; i < str.size(); i++) {
        if (str[i] == '(') {
            s.push('(');  // 左括号入栈
        } else if (str[i] == ')') {
            if (s.empty()) {
                // 栈空了还遇到右括号，说明右括号多了
                ok = false;
                break;
            }
            s.pop();  // 配对成功，弹出一个左括号
        }
    }

    // 如果到最后栈不为空，说明左括号多了
    if (!s.empty()) {
        ok = false;
    }

    if (ok) {
        cout << "括号匹配正确！" << endl;
    } else {
        cout << "括号匹配错误！" << endl;
    }

    return 0;
}
```

**测试用例**：

| 输入 | 输出 | 说明 |
|------|------|------|
| `(())()` | 括号匹配正确！ | 每个左括号都找到了配对 |
| `(()` | 括号匹配错误！ | 多了一个左括号 |
| `())` | 括号匹配错误！ | 多了一个右括号 |
| `()()()` | 括号匹配正确！ | 三对括号，完美配对 |

**逐步模拟（以 `(())()` 为例）**：

| 读到字符 | 操作 | 栈内容（底→顶） |
|----------|------|-----------------|
| `(` | 入栈 | `(` |
| `(` | 入栈 | `( (` |
| `)` | 配对，弹出 | `(` |
| `)` | 配对，弹出 | *空* |
| `(` | 入栈 | `(` |
| `)` | 配对，弹出 | *空* |

最终栈为空 → 匹配正确 ✅

### 例题 4：模拟排队过程（队列的应用，进阶）

**题目**：有 n 个人排队买冰淇淋，编号 1~n。每个人买完需要 1 分钟。但如果某人的编号是 3 的倍数，他会买完后再排到队尾重新排一次（太好吃了想再买一个）。问最终所有人的离开顺序。

**讲解思路**：

> 这就像真实排队！用队列模拟：每次让队首的人买完离开。但如果他的编号是 3 的倍数（比如 3 号、6 号），他离开后又跑去队尾重新排队。不过第二次买完就真的走了，不会再排第三次。

```cpp
#include <iostream>
#include <queue>
using namespace std;

int main() {
    int n;
    cout << "请输入排队人数: ";
    cin >> n;

    queue<int> q;           // 排队队列，存编号
    int bought[101] = {0};  // 记录每个人已经买了几次

    // 所有人排队
    for (int i = 1; i <= n; i++) {
        q.push(i);
    }

    cout << "离开顺序: ";
    while (!q.empty()) {
        int person = q.front();  // 看看队首是谁
        q.pop();                 // 离开队首

        bought[person]++;  // 买了一次

        if (person % 3 == 0 && bought[person] == 1) {
            // 编号是3的倍数，且是第一次买，重新排队
            q.push(person);
        } else {
            // 真正离开
            cout << person << " ";
        }
    }
    cout << endl;

    return 0;
}
```

**测试（n = 6）**：

初始队列：`1 2 3 4 5 6`

| 步骤 | 队首离开 | 操作 | 队列状态 | 输出 |
|------|---------|------|---------|------|
| 1 | 1 | 离开 | `2 3 4 5 6` | 1 |
| 2 | 2 | 离开 | `3 4 5 6` | 2 |
| 3 | 3 | 3是3的倍数，第1次→重新排队 | `4 5 6 3` | — |
| 4 | 4 | 离开 | `5 6 3` | 4 |
| 5 | 5 | 离开 | `6 3` | 5 |
| 6 | 6 | 6是3的倍数，第1次→重新排队 | `3 6` | — |
| 7 | 3 | 第2次，离开 | `6` | 3 |
| 8 | 6 | 第2次，离开 | *空* | 6 |

**最终输出**：`1 2 4 5 3 6`

## 七、常见错误与应对

### 错误 1：对空栈/空队列调用 `top()`、`front()` 或 `pop()`

```cpp
stack<int> s;
cout << s.top();  // ❌ 运行时错误！栈是空的，没有栈顶
```

**应对**：养成好习惯——操作前先检查 `empty()`。

```cpp
if (!s.empty()) {
    cout << s.top();
}
```

> 👨‍🏫 **老师说**：就像你不能从空盘子堆上拿盘子一样，先看看有没有盘子！

---

### 错误 2：以为 `pop()` 会返回被删除的值

```cpp
stack<int> s;
s.push(10);
int x = s.pop();  // ❌ 编译错误！pop() 返回 void，不返回值
```

**应对**：先用 `top()` 取值，再 `pop()` 删除。

```cpp
int x = s.top();  // 先看
s.pop();           // 再删
```

> 👨‍🏫 **老师说**：`top()` 是"看一眼"，`pop()` 是"扔掉"。想知道扔掉的是什么，得先看一眼记住，再扔！

---

### 错误 3：混淆栈的 `top()` 和队列的 `front()`

```cpp
queue<int> q;
q.push(1);
cout << q.top();  // ❌ 编译错误！队列没有 top()，要用 front()
```

**应对**：记忆口诀——**"栈看顶 top，队看头 front"**。

---

### 错误 4：忘记写头文件

```cpp
// 忘记 #include <stack>
stack<int> s;  // ❌ 编译错误
```

**应对**：用栈就 `#include <stack>`，用队列就 `#include <queue>`。建议写完 `#include` 后立刻编译一次确认。

---

### 错误 5：括号匹配时只检查了栈空，忘记检查多余左括号

```cpp
// 只在循环里检查了"右括号多了"的情况
// 循环结束后忘记判断栈是否为空
```

**应对**：循环结束后一定要加 `if (!s.empty()) ok = false;`，否则 `(((` 这样的输入会被误判为正确。
