## 四、讲稿提纲

### 4.1 情境导入（5 min）——快递单的故事

> 同学们，你们收过快递吗？快递小哥手上有一张**快递单**，上面写了什么？
>
> —— 收件人姓名、电话、地址、物品名称……
>
> 这些信息**类型不一样**：姓名是文字，电话是数字，地址也是文字。可它们都**属于同一个快递**，被印在**同一张单子**上。
>
> 在 C++ 里，我们以前学的 `int`、`double`、`string` 都是**单个数据**。如果我要记录一个同学的信息——姓名、年龄、成绩——用三个单独的变量可以，但如果有 50 个同学呢？
>
> 今天我们学一个新本领：**结构体（struct）**——就像一张"信息卡"，把多个不同类型的数据**打包**装在一起！

### 4.2 讲解 A：struct 定义 & 成员访问（15 min）

**核心比喻：学生证**

> 想象你手上有一张**学生证**，上面有：
> - 姓名（string）
> - 年龄（int）
> - 成绩（double）
>
> 一张学生证 = 一个结构体变量
> 学生证上的每一栏 = 一个**成员**

**板书 / PPT 展示语法：**

```
定义模板（告诉电脑学生证长什么样）：

struct 结构体名 {
    类型 成员名1;
    类型 成员名2;
    ...
};          ← 注意这里有分号！
```

**现场演示代码：**

```cpp
#include <bits/stdc++.h>
using namespace std;

// 第一步：定义结构体（设计学生证的模板）
struct Student {
    string name;   // 姓名
    int age;       // 年龄
    double score;  // 成绩
};  // ← 千万别忘了这个分号！

int main() {
    // 第二步：声明一个结构体变量（发一张学生证）
    Student a;

    // 第三步：给成员赋值（在学生证上填信息）
    a.name = "xiaoming";
    a.age = 10;
    a.score = 95.5;

    // 第四步：用点号读取成员
    cout << a.name << " " << a.age << " " << a.score << endl;

    return 0;
}
```

**教学口诀：**
> **"先设计模板，再发卡填写，点号来读取。"**

**强调点：**
- `struct` 定义写在 `main` 函数**外面**（全局）。
- 结构体名首字母**大写**（好习惯，不强制）。
- 大括号后面的**分号**是最常见的错误，一定要记住！

### 4.3 练一练 A（10 min）

> 请你设计一个结构体 `Book`，包含：书名（string）、价格（double）、页数（int）。
> 然后声明一个变量，填上你喜欢的一本书的信息，输出出来。

教师巡视，重点关注：分号是否遗漏、点号使用是否正确。

### 4.4 讲解 B：结构体数组（10 min）

**过渡语：**

> 一张学生证记录一个人。如果我们班有 5 个人参加比赛，需要 5 张学生证——怎么办？
>
> **结构体数组**！就像把 5 张学生证**排成一排、编上号**，跟 `int` 数组一模一样的道理。

**板书 / PPT：**

```
Student stu[5];       // 5 张学生证

stu[0].name = "A";    // 第 0 张
stu[0].age = 10;
stu[1].name = "B";    // 第 1 张
...
```

**现场演示：用循环录入 & 输出**

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Student {
    string name;
    int age;
    int score;
};

int main() {
    int n;
    cin >> n;
    Student stu[105];  // 开大一点，竞赛好习惯

    for (int i = 0; i < n; i++) {
        cin >> stu[i].name >> stu[i].age >> stu[i].score;
    }

    for (int i = 0; i < n; i++) {
        cout << stu[i].name << " " << stu[i].age << " " << stu[i].score << endl;
    }

    return 0;
}
```

**类比强化：**

> `int a[100]` —— 100 个格子，每格放一个整数。
> `Student stu[100]` —— 100 个格子，每格放一张学生证（里面有姓名、年龄、成绩）。

### 4.5 练一练 B（10 min）

> 输入 n 个水果的信息（名称 string、重量 double），然后输出重量最大的水果的名称和重量。
>
> 提示：遍历比较，和以前找最大值一模一样，只是现在比较的是 `fruit[i].weight`。

### 4.6 讲解 C：结构体 + sort 排序（15 min）

**过渡语：**

> 我们以前学过 `sort` 排序 `int` 数组。现在数组里装的不是一个个数字，而是一张张"学生证"——怎么排？
>
> 比如老师说："按成绩从高到低排名。"电脑不知道该看哪一栏，所以我们要**自己写一个比较函数**，告诉电脑排序规则。

**板书关键代码：**

```cpp
// 比较函数：告诉 sort "谁排前面"
bool cmp(Student a, Student b) {
    return a.score > b.score;  // 成绩高的排前面（降序）
}

// 使用
sort(stu, stu + n, cmp);
```

**教学口诀：**
> **"写个 cmp，return 谁在前。"**

**扩展：两个关键字排序（简单提一下）**

> 如果成绩相同，按姓名字典序从小到大排呢？

```cpp
bool cmp(Student a, Student b) {
    if (a.score != b.score) return a.score > b.score;  // 先按成绩降序
    return a.name < b.name;  // 成绩相同，按姓名升序
}
```

## 五、课堂例题

### 例题 1：个人名片（入门热身）

**题目：** 定义一个结构体 `Person`，包含姓名（string）和年龄（int）。输入一个人的姓名和年龄，输出 `"xxx is yy years old."`。

**讲解思路：** 最简单的结构体使用，帮助学生跑通"定义 → 声明 → 赋值 → 访问"的完整流程。

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Person {
    string name;
    int age;
};

int main() {
    Person p;
    cin >> p.name >> p.age;
    cout << p.name << " is " << p.age << " years old." << endl;
    return 0;
}
```

**教学话术：**
> 这道题很简单对不对？但是它帮你走完了结构体的"四步流程"。记住这个套路，后面的题都是在这个基础上变花样。

---

### 例题 2：全班花名册（结构体数组）

**题目：** 输入 n 个学生的姓名和成绩，输出所有成绩 >= 90 分的学生姓名。

**讲解思路：** 结构体数组 + 遍历筛选，复习 `if` 判断。

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Student {
    string name;
    int score;
};

int main() {
    int n;
    cin >> n;
    Student stu[105];

    for (int i = 0; i < n; i++) {
        cin >> stu[i].name >> stu[i].score;
    }

    cout << "Honor Roll:" << endl;
    for (int i = 0; i < n; i++) {
        if (stu[i].score >= 90) {
            cout << stu[i].name << endl;
        }
    }

    return 0;
}
```

**教学话术：**
> 以前我们用两个数组 `name[i]` 和 `score[i]` 也能做。但如果还要加年龄、班级、电话呢？数组越来越多，容易搞混。结构体把它们**打包在一起**，`stu[i]` 就是一个完整的人，清清楚楚。

---

### 例题 3：成绩排名（结构体 + sort）

**题目：** 输入 n 个学生的姓名和成绩，按成绩从高到低排序输出。如果成绩相同，按姓名字典序从小到大排。

**讲解思路：** 这是结构体最经典的应用场景——CSP-J 中大量排序题都需要这个技巧。

```cpp
#include <bits/stdc++.h>
using namespace std;

struct Student {
    string name;
    int score;
};

bool cmp(Student a, Student b) {
    if (a.score != b.score) return a.score > b.score;  // 成绩高的在前
    return a.name < b.name;  // 成绩相同，姓名字典序小的在前
}

int main() {
    int n;
    cin >> n;
    Student stu[105];

    for (int i = 0; i < n; i++) {
        cin >> stu[i].name >> stu[i].score;
    }

    sort(stu, stu + n, cmp);

    for (int i = 0; i < n; i++) {
        cout << stu[i].name << " " << stu[i].score << endl;
    }

    return 0;
}
```

**教学话术：**
> `cmp` 函数就像你当裁判：两个同学站在你面前，你决定谁排前面。先比成绩，成绩不同就高分在前；成绩一样再比名字。`sort` 每次要比较两个人时，就来问你这个"裁判函数"。

## 七、常见错误与应对

| 序号 | 常见错误 | 错误表现 | 应对方法 |
|:----:|---------|---------|---------|
| 1 | **struct 大括号后漏分号** | 编译报一大堆莫名其妙的错误 | 口诀："struct 结尾加分号，忘了编译全报错。"板书时故意漏掉让学生找 |
| 2 | **把 struct 定义写在 main 里面** | 有时能编译，但作用域受限，cmp 函数用不了 | 强调：struct 定义写在 main **外面**，和函数平级 |
| 3 | **访问成员忘记用点号** | 写成 `stu name` 或 `stu[i]name` | 提醒：点号是"翻开学生证看某一栏"的动作 |
| 4 | **结构体数组下标和成员搞混** | 写成 `stu.name[i]` 而不是 `stu[i].name` | 口诀："先选第几张卡 `[i]`，再看哪一栏 `.name`" |
| 5 | **cmp 函数返回值写反** | 想降序写成 `a.score < b.score` | 记忆技巧："return 后面写的是**你希望谁在前面**的条件" |
| 6 | **sort 忘记传 cmp** | `sort(stu, stu+n)` 无法编译（结构体没有默认 `<`） | 提醒：int 有默认排序，结构体没有，必须带裁判 |
