# 第23课 高精度加法

## 一、课程信息

| 项目 | 内容 |
|------|------|
| 课次 | 第 23 课 / 共 40 课 |
| 主题 | 高精度加法 |
| 前置知识 | 数组、循环、模拟法 |
| 课时 | 100 分钟 |

### 教学目标

1. **理解高精度的必要性**：能说出 `int` / `long long` 的存储上限，并举例说明哪些场景需要高精度。
2. **掌握高精度加法核心流程**：能独立写出"字符串读入 → 倒序存数组 → 逐位相加处理进位 → 去前导零倒序输出"的完整代码。
3. **通过课堂练习验证**：能在课堂上独立完成至少 1 道高精度加法题并通过评测。

---

## 二、教学重点与难点

### 重点

- 用数组模拟竖式加法的整体思路（拆位 → 逐位加 → 进位 → 输出）。
- 字符串转数组时的**倒序存储**技巧。
- 逐位相加时**进位的处理**。

### 难点

- 为什么要倒序存储（个位放 `a[0]`），而不是正序存储。
- 最高位产生进位后，结果位数比两个加数都多一位的情况。
- 输出时去掉前导零、再倒序输出的细节。

>
> 今天我们要学一个超级厉害的本领——**高精度加法**。学会了它，哪怕数字有一万位，我们也能算得清清楚楚！

---

### 4.2 讲解①：核心思路（5′–20′）

#### （1）生活比喻：竖式加法

> 同学们想一想，你们小学一年级就会做竖式加法了对不对？比如：
>
> ```
>     3 8 6
>   + 4 7 5
>   ------
> ```
>
> 你们是怎么算的？——**从个位开始，一位一位往左加**，满十进一。
>
> 高精度加法就是让电脑**模拟你做竖式的过程**！

#### （2）搬家比喻：为什么用数组

> 一个 100 位的大数字，就像一条长长的火车🚂，有 100 节车厢。
>
> `int` 和 `long long` 就像一个小盒子，只能装 10 节车厢的东西——装不下！
>
> 那怎么办呢？我们找 100 个小格子（数组），每个格子放一位数字。
>
> | 格子编号 | 0 | 1 | 2 | 3 | ... |
> |---------|---|---|---|---|-----|
> | 存的数字 | 6 | 8 | 3 | — | ... |
>
> 这就像把火车拆开来，每节车厢单独停在一个停车位上。

#### （3）为什么倒序存？

> 注意看，上面表格里 386 是**倒着存**的：格子 0 放个位 6，格子 1 放十位 8，格子 2 放百位 3。
>
> 为什么要倒着呢？因为我们做竖式的时候是**从个位开始加**的——如果个位在 `a[0]`，我们的循环就可以从 `i = 0` 开始，多方便呀！
>
> 而且，如果最高位产生进位，数字会多一位——倒序存的话，只要在数组后面多加一格就行了，不用把前面所有数字都搬家。

#### （4）完整流程（板书/PPT）

```
第 1 步：字符串读入两个大数 s1, s2
第 2 步：把字符串倒序转成数组 a[], b[]（字符 '0'~'9' 变成数字 0~9）
第 3 步：从 i=0 开始逐位相加：c[i] = a[i] + b[i] + 进位
第 4 步：处理进位：如果 c[i] >= 10，就 c[i] -= 10，进位 = 1
第 5 步：去掉前导零，倒序输出 c[]
```

## 五、课堂例题

### 例题 1：基础高精度加法（标准模板）

**题目**：输入两个非负整数 a 和 b（位数可达 500 位），输出 a + b。

**讲解思路**：

1. 两个数都非常大，不能用 `int` / `long long`，必须用字符串读入。
2. 字符串倒序转成整数数组，便于从低位开始加。
3. 逐位相加，处理进位。
4. 去前导零，倒序输出。

**完整代码**：

```cpp
#include <iostream>
#include <cstring>
using namespace std;

int main() {
    // 第 1 步：字符串读入
    string s1, s2;
    cin >> s1 >> s2;

    // 准备数组：a 存第一个数，b 存第二个数，c 存结果
    // 数组开大一点，510 够用了（最多 500 位 + 进位）
    int a[510] = {0};
    int b[510] = {0};
    int c[510] = {0};

    int len1 = s1.size();  // 第一个数的位数
    int len2 = s2.size();  // 第二个数的位数

    // 第 2 步：字符串倒序存入数组
    // s1 = "386"，那么 s1[0]='3', s1[1]='8', s1[2]='6'
    // 我们要让 a[0]=6, a[1]=8, a[2]=3（个位在前）
    for (int i = 0; i < len1; i++) {
        a[i] = s1[len1 - 1 - i] - '0';
    }
    for (int i = 0; i < len2; i++) {
        b[i] = s2[len2 - 1 - i] - '0';
    }

    // 第 3 步：逐位相加
    int len = max(len1, len2);  // 取较长的位数
    int carry = 0;              // 进位，一开始是 0

    for (int i = 0; i < len; i++) {
        int sum = a[i] + b[i] + carry;  // 当前位的和 = 两个数的这一位 + 进位
        c[i] = sum % 10;                // 当前位的结果（取个位）
        carry = sum / 10;               // 新的进位（取十位）
    }

    // 第 4 步：如果最高位还有进位，别忘了！
    if (carry > 0) {
        c[len] = carry;
        len++;  // 结果多了一位
    }

    // 第 5 步：倒序输出
    for (int i = len - 1; i >= 0; i--) {
        cout << c[i];
    }
    cout << endl;

    return 0;
}
```

**课堂演示**：

```
输入：
9876
2345
输出：
12221
```

```
输入：
999
1
输出：
1000
```

> 🎯 **老师强调**：
> - `s1[len1 - 1 - i] - '0'` 这里的 `- '0'` 是把字符变成数字，还记得 ASCII 码吗？字符 `'3'` 的编号是 51，数字 `3` 是 `51 - 48 = 3`，而 `'0'` 的编号就是 48。所以 `'3' - '0' = 3`。
> - `carry`（进位）就像是你做竖式时写在上面的那个小小的 "1"。
> - `sum % 10` 留下个位写在当前格子，`sum / 10` 就是要进到下一位的数。

---

### 例题 2：两个数位数差距很大

**题目**：计算 `100000000000000000000`（1 后面 20 个 0）加 `1`。

**讲解思路**：

> 这道题考验的是：两个数位数差很多的时候，程序还能不能正确运行？
>
> 答案是：当然可以！因为我们数组初始化为全 0，短的那个数后面的高位自动是 0，加上去不影响结果。

**课堂运行**：

```
输入：
100000000000000000000
1
输出：
100000000000000000001
```

> 老师引导学生思考：
> - 如果用 `long long` 算这道题会怎样？（答：`long long` 存不下 21 位的数，会出错！）
> - 但我们的高精度代码轻轻松松就搞定了，是不是很厉害？👏

---

### 例题 3：连续进位的情况

**题目**：计算 `999999` + `1`。

**讲解思路**：

> 这道题是"连续进位"，从个位一直进到最高位还要再进一位。就像多米诺骨牌一样，一个接一个倒下去。

**课堂推演（和学生一起走一遍循环）**：

```
i=0: a[0]=9, b[0]=1, carry=0 → sum=10, c[0]=0, carry=1
i=1: a[1]=9, b[1]=0, carry=1 → sum=10, c[1]=0, carry=1
i=2: a[2]=9, b[2]=0, carry=1 → sum=10, c[2]=0, carry=1
i=3: a[3]=9, b[3]=0, carry=1 → sum=10, c[3]=0, carry=1
i=4: a[4]=9, b[4]=0, carry=1 → sum=10, c[4]=0, carry=1
i=5: a[5]=9, b[5]=0, carry=1 → sum=10, c[5]=0, carry=1
循环结束后 carry=1，所以 c[6]=1，len 变成 7。
倒序输出：1000000 ✓
```

> 🎯 **老师强调**：循环结束后**一定要检查 carry**！这是很多同学容易忘掉的地方。忘了的话，999999 + 1 就只会输出 000000，结果完全错了！

## 七、常见错误与应对

### ❌ 错误 1：忘记倒序存储

```cpp
// 错误写法
for (int i = 0; i < len1; i++) {
    a[i] = s1[i] - '0';  // 这样 a[0] 存的是最高位！
}
```

**后果**：加法从最高位开始加了，进位方向完全反了，结果一塌糊涂。

**应对**：提醒学生记住口诀——**"个位打头，倒着存"**。可以让学生在代码旁边写注释 `// 倒序存储：a[0] 是个位`。

---

### ❌ 错误 2：循环结束后忘记处理最后的进位

```cpp
// 忘记了这段！
if (carry > 0) {
    c[len] = carry;
    len++;
}
```

**后果**：`999 + 1` 会输出 `000` 而不是 `1000`。

**应对**：告诉学生："循环结束了，但进位可能还在'口袋'里没放出来呢！一定要**最后再检查一次进位**。"

---

### ❌ 错误 3：字符转数字时忘记 `- '0'`

```cpp
a[i] = s1[len1 - 1 - i];  // 少了 - '0'！
```

**后果**：`a[i]` 存的是字符的 ASCII 码（比如 48~57），不是 0~9，加出来全是乱七八糟的大数。

**应对**：提醒学生——字符串里拿出来的是**字符**，不是数字。`'3'` 不等于 `3`！要减去 `'0'` 才能变成数字。

---

### ❌ 错误 4：数组开得太小

```cpp
int a[100];  // 如果数字有 500 位就越界了！
```

**后果**：数组越界，程序可能崩溃或输出奇怪的结果。

**应对**：养成习惯——**数组多开一点**。题目说最多 500 位，我们就开 510，留点余量。

---

### ❌ 错误 5：没有初始化数组为 0

```cpp
int a[510];  // 没有 = {0}
```

**后果**：数组里可能有"垃圾值"，导致结果出错。

**应对**：定义数组时加上 `= {0}`，养成好习惯。或者用 `memset(a, 0, sizeof(a))`。

---

### ❌ 错误 6：输出前导零

比如计算 `100 + 200 = 300`，如果处理不当可能输出 `0300`。

**应对**：在本课的标准代码中，因为我们用 `len` 记录了实际位数，倒序输出时从 `len-1` 开始，一般不会有前导零。但如果学生自己改了代码结构，提醒他们注意去掉前导零。特殊情况：两个数都是 0 时，结果应该输出 `0` 而不是空。
