### 重点

- 递归的概念：函数在自己的身体里调用自己。
- 递归三要素的识别与编写。
- 用调用栈图理解递归的"去"和"回"两个过程。

### 难点

- 理解"函数还没执行完就又调用了自己"——执行流程的跳转。
- 终止条件的设计：什么时候该停下来？
- 从"循环思维"转向"递归思维"：把大问题拆成"做一点 + 剩下的交给自己"。

## 四、讲稿提纲

### 4.1 导入——俄罗斯套娃的故事（5 min）

> 同学们，老师今天带来了一个"俄罗斯套娃"。你们玩过吗？
>
> 打开最外面的大娃娃——哇，里面还有一个小一点的娃娃！
> 再打开——里面还有一个更小的！
> 再打开——又有一个！
> 一直打开……直到最后那个**最小的实心娃娃**，打不开了，就停下来了。
>
> 这就是我们今天要学的新本领——**递归**。
>
> 递归就像套娃：**每一层做的事情都一样**（打开娃娃），但**娃娃越来越小**（问题越来越简单），直到碰到**最小的那个娃娃**（终止条件），就不用再打开了。

> 再想一个画面：你站在两面平行的镜子中间，看一面镜子——里面有你，你里面还有你，还有你……无穷无尽！如果没有"最小的娃娃"来喊停，递归也会像这面镜子一样停不下来，电脑就崩溃了。

### 4.2 递归的概念（5 min）

> 在编程里，**递归**就是——一个函数在自己身体里面**调用自己**。
>
> 听起来很奇怪对不对？函数怎么能调自己呢？我们来看一个最简单的例子。

```
void 打开套娃(int 娃娃编号) {
    if (娃娃编号 == 最小的) {
        cout << "到底了，不用再打开了！" << endl;
        return;  // 停下来！
    }
    cout << "打开第" << 娃娃编号 << "个娃娃" << endl;
    打开套娃(娃娃编号 + 1);  // 打开里面那个更小的
}
```

> 看！`打开套娃` 这个函数里面，又调用了 `打开套娃`，只不过娃娃编号变大了（娃娃变小了）。这就是递归！

### 4.3 递归三要素（10 min，重点板书）

> 写递归函数，要记住三个宝贝，一个都不能少：

| 编号 | 要素 | 套娃比喻 | 作用 |
|------|------|----------|------|
| ① | **做什么**（当前这一层要干的事） | 打开眼前这个娃娃 | 处理当前问题 |
| ② | **递归调用**（把问题缩小，交给自己） | 去打开里面更小的娃娃 | 让问题越来越简单 |
| ③ | **终止条件**（最小的娃娃，该停了） | 碰到实心的小娃娃就停 | 防止无穷无尽 |

> **口诀**：做事、缩小、喊停——三步写递归！

### 4.4 倒计时例子——从 N 数到 1（板书+画图）

> 想象老师说："从5开始倒数！"
> 你说：5！然后让旁边的同学从4开始倒数。
> 旁边的同学说：4！又让他旁边的从3开始倒数。
> ……一直到有人数到1，说"1！结束了！"就停下来。

**画调用栈图**（板书核心，慢慢画，每一步停下来问学生）：

```
countdown(5)  →  输出5，调用 countdown(4)
  countdown(4)  →  输出4，调用 countdown(3)
    countdown(3)  →  输出3，调用 countdown(2)
      countdown(2)  →  输出2，调用 countdown(1)
        countdown(1)  →  输出1，n==1 到底了，return！
      countdown(2) 回来了，结束
    countdown(3) 回来了，结束
  countdown(4) 回来了，结束
countdown(5) 回来了，结束
```

> 看到了吗？像搭积木一样，一层一层叠上去，到底之后再一层一层撤回来。这个"叠积木"的过程，计算机里叫做**调用栈**。

### 4.5 没有终止条件会怎样？（演示+讨论）

> 如果最小的套娃也能打开，里面还有更小的，永远打不完——会怎样？
>
> 电脑的调用栈就像一个有限高度的积木塔。如果一直叠叠叠，叠到天花板——"轰"一声塌了！
> 这就叫**栈溢出**（Stack Overflow），电脑会报错崩溃。
>
> 所以，**终止条件**是递归的安全网，绝对不能忘！

（现场演示一个没有终止条件的递归程序，让学生看到报错信息。）

### 4.6 求和 1+2+…+N（讲解②核心）

> 老师问你：1+2+3+4+5 等于多少？
>
> 你可以这样想：
> - 1+2+3+4+5 = **5** + (1+2+3+4)
> - 1+2+3+4 = **4** + (1+2+3)
> - 1+2+3 = **3** + (1+2)
> - 1+2 = **2** + (1)
> - 1 就是 1，不用再拆了！（终止条件）
>
> 发现了吗？`sum(n) = n + sum(n-1)`，这就是递归公式！
> 最小的情况是 `sum(1) = 1`，这就是终止条件！

## 五、课堂例题

### 例题 1：倒计时（从 N 数到 1）⭐

**题目**：输入一个正整数 N（1 ≤ N ≤ 100），用递归函数输出从 N 到 1 的倒计时，每个数字占一行。

**讲解思路**：
- **做什么**：输出当前的数字 n。
- **递归调用**：调用自己，参数变成 n-1（问题缩小了）。
- **终止条件**：当 n == 0 时，不再输出，直接 return。

```cpp
#include <iostream>
using namespace std;

void countdown(int n) {
    if (n == 0) {       // ③ 终止条件：数到0就停
        return;
    }
    cout << n << endl;   // ① 做什么：输出当前数字
    countdown(n - 1);    // ② 递归调用：数字缩小1
}

int main() {
    int n;
    cin >> n;
    countdown(n);
    return 0;
}
```

**输入输出示例**：

```
输入：5
输出：
5
4
3
2
1
```

**板书：调用栈图**

```
main() 调用 countdown(5)
│
├─ countdown(5): 输出5, 调用 countdown(4)
│  ├─ countdown(4): 输出4, 调用 countdown(3)
│  │  ├─ countdown(3): 输出3, 调用 countdown(2)
│  │  │  ├─ countdown(2): 输出2, 调用 countdown(1)
│  │  │  │  ├─ countdown(1): 输出1, 调用 countdown(0)
│  │  │  │  │  └─ countdown(0): n==0, return!
│  │  │  │  └─ 返回
│  │  │  └─ 返回
│  │  └─ 返回
│  └─ 返回
└─ 返回
```

### 例题 2：递归求和 1+2+…+N ⭐⭐

**题目**：输入正整数 N（1 ≤ N ≤ 1000），用递归函数求 1+2+3+…+N 的值并输出。

**讲解思路**：
- **做什么**：当前这一层负责把 n 加上去。
- **递归调用**：`sum(n-1)` 算出 1+2+…+(n-1) 的结果。
- **终止条件**：`sum(1) = 1`，只剩一个数，直接返回。
- **递归公式**：`sum(n) = n + sum(n-1)`

```cpp
#include <iostream>
using namespace std;

int sum(int n) {
    if (n == 1) {           // ③ 终止条件：只有1个数
        return 1;
    }
    return n + sum(n - 1);  // ① 做什么 + ② 递归调用
}

int main() {
    int n;
    cin >> n;
    cout << sum(n) << endl;
    return 0;
}
```

**输入输出示例**：

```
输入：5
输出：15
```

**板书：展开过程（带返回值）**

```
sum(5) = 5 + sum(4)
             sum(4) = 4 + sum(3)
                          sum(3) = 3 + sum(2)
                                       sum(2) = 2 + sum(1)
                                                    sum(1) = 1  ← 到底了！
                                       sum(2) = 2 + 1 = 3      ← 回来了
                          sum(3) = 3 + 3 = 6
             sum(4) = 4 + 6 = 10
sum(5) = 5 + 10 = 15  ✓
```

> 重点引导学生观察：递归是"先往下钻，到底之后再一层一层把答案带回来"。

### 例题 3：正序输出每一位数字 ⭐⭐

**题目**：输入一个正整数 N（1 ≤ N ≤ 1000000000），用递归函数从高位到低位依次输出它的每一位数字，每位数字之间用空格隔开。

**讲解思路**：
- 如果用循环，从低位取容易（% 10），但从高位开始输出就麻烦了。
- 递归的妙处：**先递归（处理前面的高位），回来之后再输出当前最低位**。
- 但这题我们换个思路：**先递归处理 n/10（去掉最后一位），再输出 n%10（最后一位）**。
- **终止条件**：n 是个位数（n < 10）时，直接输出。

```cpp
#include <iostream>
using namespace std;

void printDigits(int n) {
    if (n < 10) {              // ③ 终止条件：只剩一位数
        cout << n;
        return;
    }
    printDigits(n / 10);       // ② 递归调用：先输出前面的高位
    cout << " " << n % 10;     // ① 做什么：输出当前最低位
}

int main() {
    int n;
    cin >> n;
    printDigits(n);
    cout << endl;
    return 0;
}
```

**输入输出示例**：

```
输入：2048
输出：2 0 4 8
```

**板书：调用栈图**

```
printDigits(2048)
  → 先调用 printDigits(204)
      → 先调用 printDigits(20)
          → 先调用 printDigits(2)
              → 2 < 10, 输出 "2", return
          → 输出 " 0"  (20 % 10 = 0)
      → 输出 " 4"  (204 % 10 = 4)
  → 输出 " 8"  (2048 % 10 = 8)

最终输出：2 0 4 8
```

> 引导学生体会：**递归调用放在输出前面**，就能实现"先处理高位，后处理低位"——这是递归的一个巧妙用法！

### 例题 4（拓展）：递归判断回文串 ⭐⭐⭐

**题目**：输入一个只含小写字母的字符串 s（长度 ≤ 100），用递归判断它是否是回文串，输出 `Yes` 或 `No`。

**讲解思路**：
- 回文串：正读反读都一样，如 "abcba"。
- 递归思路：比较**第一个字符**和**最后一个字符**——
  - 如果不一样 → 不是回文，直接返回 false。
  - 如果一样 → 把"头尾去掉"，剩下的部分交给递归去判断。
- **终止条件**：字符串长度 ≤ 1 时，一定是回文（空串或单个字符）。

```cpp
#include <iostream>
#include <string>
using namespace std;

bool isPalindrome(string s, int left, int right) {
    if (left >= right) {        // ③ 终止条件：检查完了
        return true;
    }
    if (s[left] != s[right]) {  // 头尾不一样，不是回文
        return false;
    }
    // ② 递归调用：去掉头尾，检查中间部分
    return isPalindrome(s, left + 1, right - 1);
}

int main() {
    string s;
    cin >> s;
    if (isPalindrome(s, 0, s.size() - 1)) {
        cout << "Yes" << endl;
    } else {
        cout << "No" << endl;
    }
    return 0;
}
```

**输入输出示例**：

```
输入：abcba
输出：Yes

输入：hello
输出：No
```

**板书：调用栈图（以 "abcba" 为例）**

```
isPalindrome("abcba", 0, 4)   a==a ✓
  isPalindrome("abcba", 1, 3) b==b ✓
    isPalindrome("abcba", 2, 2) left>=right → return true!
  → return true
→ return true
```

> 这道题展示了递归在字符串问题上的妙用。和之前学的字符串知识结合起来了！

### 练习 1：正数倒数（热身）⭐

**题目**：输入正整数 N（1 ≤ N ≤ 50），用递归函数输出从 1 到 N 的所有整数（正序），每个数字占一行。

**提示**：和倒计时很像，但输出顺序反了。想一想：是先输出再递归，还是先递归再输出？

**参考答案**：

```cpp
#include <iostream>
using namespace std;

void countUp(int n) {
    if (n == 0) return;
    countUp(n - 1);     // 先递归（处理前面的小数）
    cout << n << endl;   // 再输出（回来时输出当前数）
}

int main() {
    int n;
    cin >> n;
    countUp(n);
    return 0;
}
```

### 练习 2：递归求 N 的阶乘 ⭐⭐

**题目**：输入正整数 N（1 ≤ N ≤ 15），用递归函数求 N! = 1 × 2 × 3 × … × N，输出结果。

**提示**：`factorial(n) = n * factorial(n-1)`，终止条件是 `factorial(1) = 1`。

**参考答案**：

```cpp
#include <iostream>
using namespace std;

long long factorial(int n) {
    if (n == 1) return 1;
    return n * factorial(n - 1);
}

int main() {
    int n;
    cin >> n;
    cout << factorial(n) << endl;
    return 0;
}
```

### 练习 3：递归求斐波那契第 N 项 ⭐⭐

**题目**：斐波那契数列：1, 1, 2, 3, 5, 8, 13, 21, …（从第3项开始，每项 = 前两项之和）。输入正整数 N（1 ≤ N ≤ 25），用递归函数求第 N 项的值。

**提示**：`fib(n) = fib(n-1) + fib(n-2)`，终止条件是 `fib(1) = 1, fib(2) = 1`。

**参考答案**：

```cpp
#include <iostream>
using namespace std;

int fib(int n) {
    if (n == 1 || n == 2) return 1;
    return fib(n - 1) + fib(n - 2);
}

int main() {
    int n;
    cin >> n;
    cout << fib(n) << endl;
    return 0;
}
```

> 教师提醒：可以让学生试试 N=40 会怎样——非常慢！为什么？因为有大量重复计算。这个问题以后学"记忆化搜索"和"动态规划"时会解决。

### 练习 4（选做挑战）：递归输出二进制 ⭐⭐⭐

**题目**：输入正整数 N（1 ≤ N ≤ 1000000），用递归函数输出它的二进制表示（无前导零）。

**提示**：思路和"正序输出每位数字"类似，把 `/10` 和 `%10` 换成 `/2` 和 `%2`。

**参考答案**：

```cpp
#include <iostream>
using namespace std;

void printBinary(int n) {

}
```

## 七、常见错误与应对

### 错误 1：忘记写终止条件

```cpp
void countdown(int n) {
    cout << n << endl;
    countdown(n - 1);  // 没有 if 判断，永远不会停！
}
```

**现象**：程序疯狂输出，然后崩溃（Segmentation Fault 或 Stack Overflow）。

**应对**：告诉学生口诀——"写递归，第一件事就是写终止条件！先写 if，再写别的。"

---

### 错误 2：终止条件写错，差了一步

```cpp
int sum(int n) {
    if (n == 0) return 0;  // 应该是 n==1 return 1（如果题目从1开始）
    return n + sum(n - 1);
}
```

**现象**：如果 n==0 return 0 其实也能正确工作（`sum(1) = 1 + sum(0) = 1 + 0 = 1`），但如果输入了负数就会栈溢出。

**应对**：让学生养成习惯——写完后用小数据（n=1, n=2）手动模拟一遍，验证终止条件是否正确。

---

### 错误 3：递归调用时问题没有缩小

```cpp
void test(int n) {
    if (n == 0) return;
    cout << n << endl;
    test(n);  // 参数没变！永远是 n，永远不会到 0！
}
```

**现象**：同一个数字无限输出，然后崩溃。

**应对**：提醒学生检查——"递归调用的参数有没有变小（或变简单）？如果没变，套娃永远一样大，打不完！"

---

### 错误 4：递归方向搞反

```cpp
void countdown(int n) {
    if (n == 0) return;
    countdown(n - 1);     // 先递归
    cout << n << endl;     // 后输出
}
// 本想输出 5 4 3 2 1，结果输出了 1 2 3 4 5
```

**现象**：输出顺序和预期相反。

**应对**：画调用栈图！让学生搞清楚"先递归再做事"和"先做事再递归"的输出顺序是反的。这恰好也是练习1的考点。

---

### 错误 5：返回值忘记接收

```cpp
int sum(int n) {
    if (n == 1) return 1;
    sum(n - 1);           // 调用了，但没有 return！
}
```

**现象**：函数返回了随机数（未定义行为）。

**应对**：有返回值的递归函数，递归调用前面一定要写 `return`。
