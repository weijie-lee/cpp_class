## 四、讲稿提纲

### 第一段（5′–20′）：阶乘递归 & 斐波那契递归

#### 4.1 复习开场（0′–5′）

> 同学们，上节课我们学了一个神奇的东西——递归！谁还记得递归的三要素？
>
> 对！**函数自己调用自己**、**每次问题变小**、**有一个终止条件（出口）**。
>
> 今天我们要用递归来解决三个经典问题，难度一个比一个大，像闯关一样！

#### 4.2 阶乘递归（5′–12′）

> **故事引入：排队领奖品**
>
> 想象一下，学校发奖品，5 个同学要排成一队。排队方式有多少种呢？
>
> - 第一个位置有 5 个人可以选
> - 第二个位置有 4 个人可以选
> - 第三个位置有 3 个人
> - ……
>
> 所以答案是 5 × 4 × 3 × 2 × 1 = 120 种。这就是 **5 的阶乘**，写成 `5!`。
>
> 现在我们换一种想法：
>
> - 想知道 5 个人怎么排？→ 先让我知道 4 个人怎么排，然后乘以 5
> - 想知道 4 个人怎么排？→ 先让我知道 3 个人怎么排，然后乘以 4
> - ……一直到 0 个人怎么排？ → 只有 1 种方式（谁都不排，就 1 种）
>
> 看！这就是递归啊！大问题拆成小问题，最后有个出口！
>
> **公式：** `n! = n × (n-1)!`，出口：`0! = 1`

**板书推演 factorial(4)：**

```
factorial(4)
= 4 * factorial(3)
= 4 * 3 * factorial(2)
= 4 * 3 * 2 * factorial(1)
= 4 * 3 * 2 * 1 * factorial(0)
= 4 * 3 * 2 * 1 * 1
= 24
```

#### 4.3 斐波那契数列（12′–20′）

> **故事引入：兔子生兔子**
>
> 很久很久以前，有一个数学家叫斐波那契，他研究兔子繁殖：
>
> - 第 1 个月：1 对小兔子
> - 第 2 个月：还是 1 对（小兔子还没长大）
> - 第 3 个月：变成 2 对（小兔子长大生了 1 对）
> - 第 4 个月：变成 3 对
> - 第 5 个月：变成 5 对
>
> 规律是什么？ **1, 1, 2, 3, 5, 8, 13, 21, …**
>
> 每个数 = 前两个数加起来！用递归来说就是：
>
> - `fib(n) = fib(n-1) + fib(n-2)`
> - 出口：`fib(1) = 1, fib(2) = 1`
>
> ⚠️ 小提示：斐波那契递归算大数字会很慢很慢，之后第38课我们学"动态规划"会有更快的方法！今天先学会递归写法就好。

### 第二段（30′–55′）：汉诺塔问题

#### 4.4 故事引入

> **古老的传说**
>
> 在印度有一座寺庙，庙里有三根金柱子（我们叫它 A、B、C），A 柱子上从小到大叠着 64 个金盘子，最小的在最上面，最大的在最下面。
>
> 和尚们的任务是：把所有盘子从 A 柱子移到 C 柱子上，但有两个规则：
>
> 1. **一次只能移一个盘子**
> 2. **大盘子绝不能放在小盘子上面**
>
> 传说如果64个盘子全部移完，世界就会毁灭……（别担心，那需要 18446744073709551615 步，就算一秒移一个也要五千多亿年呢！😄）
>
> 今天我们不搞 64 个，先从 1 个、2 个、3 个开始！

#### 4.5 实物演示（准备 3 本不同大小的书 / 3 个不同大小的杯子）

> **请一位同学上台来试试！**
>
> 先试 **1 个盘**（1 本书）：直接从 A 搬到 C。1 步搞定！
>
> 再试 **2 个盘**：
>
> 1. 小盘 A→B
> 2. 大盘 A→C
> 3. 小盘 B→C
>
> 一共 3 步！
>
> 再试 **3 个盘**（全班一起喊步骤）：
>
> 1. 小盘 A→C
> 2. 中盘 A→B
> 3. 小盘 C→B
> 4. 大盘 A→C ← 🎉 最大的盘到位了！
> 5. 小盘 B→A
> 6. 中盘 B→C
> 7. 小盘 A→C
>
> 一共 7 步！谁发现规律了？ 1步、3步、7步…… **每次都是上一次的两倍再加1！**

#### 4.6 递归思路讲解

> 3 个盘看起来很复杂对不对？但如果我们用递归的思想，就变得超级简单——只需要 **三步**！
>
> 想象你是一个搬家公司的老板，要把 n 个盘子从 A 搬到 C：
>
> **第一步：** 先请你的"副手"（递归）把上面 n-1 个小盘子从 A 搬到 B（C 当中转站）
>
> **第二步：** 你亲自把最底下的大盘子从 A 搬到 C（这是你唯一要干的活！）
>
> **第三步：** 再请你的"副手"（递归）把那 n-1 个小盘子从 B 搬到 C（A 当中转站）
>
> **"等等，副手怎么知道怎么搬 n-1 个盘子？"**
>
> 副手也用同样的方法！他也请自己的副手搬 n-2 个……一直到只剩 1 个盘子的时候，直接搬就行了！这就是递归的出口！
>
> 📌 **记住这个口诀：搬上面 → 移最大 → 搬回来**

#### 4.7 代码讲解

> 注意看代码里的参数：`from`（从哪来）、`aux`（中转站）、`to`（到哪去）
>
> 每次递归调用的时候，这三个角色会互相交换位置——这是汉诺塔代码最巧妙的地方！

### 第三段（75′–85′）：递归 vs 循环

> 有同学会问：用 for 循环也能算阶乘啊，为什么要用递归？
>
> 好问题！我们来对比一下：
>
> | | 递归 | 循环 |
> |---|---|---|
> | 像什么 | 套娃，自己装自己 | 跑圈，一圈又一圈 |
> | 阶乘 | ✅ 好写 | ✅ 也好写 |
> | 汉诺塔 | ✅ 非常简洁 | ❌ 循环写起来很痛苦 |
> | 速度 | 有时候慢（比如斐波那契） | 通常快一些 |
> | 什么时候用递归？ | 问题能自然地拆成"同样结构的小问题"时 | 简单重复的时候 |
>
> **总结：能用循环就循环，但有些问题（比如汉诺塔），递归是最自然、最优雅的解法！**

## 五、课堂例题

### 例题 1：阶乘递归（入门级 ⭐）

**题目：** 输入一个非负整数 n（0 ≤ n ≤ 20），输出 n 的阶乘。

**讲解思路：**

1. 先和学生一起回忆阶乘的定义：`5! = 5×4×3×2×1`
2. 用递归的眼光重新看：`5! = 5 × 4!`，`4! = 4 × 3!` ……
3. 出口是什么？`0! = 1`（0 个人排队只有 1 种方式——就是没有人排）
4. 带学生手动展开 `factorial(4)` 的调用链
5. 强调返回值的类型用 `long long`（20! 非常大）

```cpp
#include <iostream>
using namespace std;

// 递归求阶乘
long long factorial(int n) {
    // 递归出口：0! = 1
    if (n == 0) {
        return 1;
    }
    // 递归公式：n! = n * (n-1)!
    return (long long)n * factorial(n - 1);
}

int main() {
    int n;
    cin >> n;
    cout << factorial(n) << endl;
    return 0;
}
```

**课堂互动：** 让学生在纸上画出 `factorial(4)` 的调用过程（画"楼梯图"），从上往下展开，再从下往上收回来。

```
factorial(4) 开始，问 factorial(3)
  factorial(3) 开始，问 factorial(2)
    factorial(2) 开始，问 factorial(1)
      factorial(1) 开始，问 factorial(0)
        factorial(0) 到达出口，返回 1
      factorial(1) 收到 1，返回 1×1 = 1
    factorial(2) 收到 1，返回 2×1 = 2
  factorial(3) 收到 2，返回 3×2 = 6
factorial(4) 收到 6，返回 4×6 = 24   ✅
```

### 例题 2：斐波那契数列递归（入门级 ⭐⭐）

**题目：** 输入一个正整数 n（1 ≤ n ≤ 30），输出斐波那契数列的第 n 项。

**讲解思路：**

1. 回忆兔子的故事：`1, 1, 2, 3, 5, 8, 13, 21, …`
2. 递归公式：`fib(n) = fib(n-1) + fib(n-2)`
3. 出口：`fib(1) = 1, fib(2) = 1`
4. 在黑板上画"树形图"展示 `fib(5)` 的调用过程
5. 让学生观察：`fib(3)` 被算了好几次！引出"递归有时候效率不高"——为后面学动态规划埋伏笔

```cpp
#include <iostream>
using namespace std;

// 递归求斐波那契第 n 项
int fib(int n) {
    // 递归出口
    if (n == 1 || n == 2) {
        return 1;
    }
    // 递归公式
    return fib(n - 1) + fib(n - 2);
}

int main() {
    int n;
    cin >> n;
    cout << fib(n) << endl;
    return 0;
}
```

**课堂互动：** 让学生试一下 `fib(40)`，感受递归变慢的过程。再试 `fib(50)`——可能要等很久甚至算不出来！让学生亲身感受"递归也有缺点"。

```
fib(5) 的调用树：

          fib(5)
         /      \
      fib(4)    fib(3)
      /    \     /    \
   fib(3) fib(2) fib(2) fib(1)
   /    \
fib(2) fib(1)

可以看到 fib(3) 被计算了 2 次，fib(2) 被计算了 3 次！
n 越大，重复越多，所以越来越慢。
```

### 例题 3：汉诺塔（核心例题 ⭐⭐⭐）

**题目：** 输入一个正整数 n（1 ≤ n ≤ 10），表示汉诺塔的盘子数量。输出把所有盘子从 A 柱移动到 C 柱的每一步操作，最后输出总步数。

**样例输入：** `3`

**样例输出：**

```
A -> C
A -> B
C -> B
A -> C
B -> A
B -> C
A -> C
Total: 7
```

**讲解思路：**

1. 用实物（三叠书）再演示一遍 3 个盘的过程
2. 引导学生发现"三步法"：
   - 步骤 1-3：把上面 2 个盘从 A 搬到 B（C 当中转）
   - 步骤 4：把最大的盘从 A 搬到 C
   - 步骤 5-7：把上面 2 个盘从 B 搬到 C（A 当中转）
3. 写代码：`hanoi(n, from, aux, to)` 四个参数
4. 带学生手动模拟 `hanoi(2, 'A', 'B', 'C')` 的调用过程

```cpp
#include <iostream>
using namespace std;

int cnt = 0;  // 记录总步数

// 汉诺塔递归
// n: 盘子数量
// from: 起始柱（盘子现在在哪）
// aux: 中转柱（借助哪根柱子）
// to: 目标柱（盘子要去哪）
void hanoi(int n, char from, char aux, char to) {
    // 递归出口：只有 1 个盘子，直接搬
    if (n == 1) {
        cout << from << " -> " << to << endl;
        cnt++;
        return;
    }

    // 第一步：把上面 n-1 个盘从 from 搬到 aux（借助 to）
    hanoi(n - 1, from, to, aux);

    // 第二步：把最大的盘从 from 搬到 to
    cout << from << " -> " << to << endl;
    cnt++;

    // 第三步：把 n-1 个盘从 aux 搬到 to（借助 from）
    hanoi(n - 1, aux, from, to);
}

int main() {
    int n;
    cin >> n;
    hanoi(n, 'A', 'B', 'C');
    cout << "Total: " << cnt << endl;
    return 0;
}
```

**重点板书：参数角色互换表**

| 调用 | from（起点） | aux（中转） | to（终点） |
|------|:---:|:---:|:---:|
| 原问题 `hanoi(n, A, B, C)` | A | B | C |
| 第一步 `hanoi(n-1, A, C, B)` | A | C | **B** |
| 第三步 `hanoi(n-1, B, A, C)` | **B** | A | C |

> 📌 口诀："第一步把终点和中转换一换，第三步把起点和中转换一换"

**课堂互动：** 让学生手动写出 `hanoi(2, 'A', 'B', 'C')` 展开过程：

```
hanoi(2, A, B, C):
  hanoi(1, A, C, B):   → 输出 A -> B
  输出 A -> C
  hanoi(1, B, A, C):   → 输出 B -> C

所以 2 个盘的输出是：
A -> B
A -> C
B -> C
共 3 步 ✅
```

### 例题 4：阶乘循环 vs 递归对比（拓展 ⭐）

**题目：** 用循环的方式再写一遍阶乘函数，对比两种写法。

**讲解思路：** 让学生看到同一个问题的两种解法，理解各自的特点。

```cpp
#include <iostream>
using namespace std;

// 方法一：递归
long long factorial_rec(int n) {
    if (n == 0) return 1;
    return (long long)n * factorial_rec(n - 1);
}

// 方法二：循环
long long factorial_loop(int n) {
    long long result = 1;
    for (int i = 1; i <= n; i++) {
        result *= i;
    }
    return result;
}

int main() {
    int n;
    cin >> n;
    cout << "递归结果：" << factorial_rec(n) << endl;
    cout << "循环结果：" << factorial_loop(n) << endl;
    return 0;
}
```

> 两个方法结果一样！但是递归的代码更"优雅"——只需要两行核心代码。而循环的代码更"直白"——一步一步乘上去。
>
> **简单问题两种都行；复杂问题（如汉诺塔）递归是王者！**

## 七、常见错误与应对

### 错误 1：忘写递归出口 → 无限递归 → 程序崩溃

```cpp
// ❌ 错误写法：没有出口！
long long factorial(int n) {
    return (long long)n * factorial(n - 1);  // n 不断减小但永远不会停
}
```

**现象：** 程序运行后卡住或直接崩溃，报 `Segmentation fault` 或 `Stack overflow`。

**应对：**

> "递归就像下楼梯，出口就是一楼。如果没有一楼，你就会一直往地下走，掉进地心啦！写递归的第一件事——**先写出口**！"

---

### 错误 2：汉诺塔参数顺序写错

```cpp
// ❌ 错误：第一步应该把 n-1 个盘搬到 aux，但 to 和 aux 写反了
hanoi(n - 1, from, aux, to);  // 这样搬到了 to 而不是 aux！
```

**应对：**

> "记住口诀：第一步——起点不变，终点和中转互换；第三步——终点不变，起点和中转互换。写完后拿 n=2 手动验一遍！"

---

### 错误 3：阶乘用 int 导致溢出

```cpp
// ❌ 错误：int 最大约 21 亿，13! = 6227020800 就溢出了
int factorial(int n) {
    if (n == 0) return 1;
    return n * factorial(n - 1);
}
```

**应对：**

> "阶乘增长特别快，像滚雪球一样越滚越大。20! 已经是一个 19 位数了！所以一定要用 `long long`。"

---

### 错误 4：斐波那契出口只写了一个

```cpp
// ❌ 错误：只写了 n==1 的出口
int fib(int n) {
    if (n == 1) return 1;
    return fib(n - 1) + fib(n - 2);  // 当 n=2 时，fib(0) 没有出口！
}
```

**应对：**

> "斐波那契数列需要**两个**出口，因为它每次要回头看**两个**小朋友。一个出口不够用！`n==1` 和 `n==2` 都要写。"

---

### 错误 5：对递归"不信任"，试图追踪每一层调用

**现象：** 学生在写汉诺塔时，总想把每一步都想清楚，越想越乱，最后放弃。

**应对：**

> "递归就像请快递小哥帮你送包裹——你只要告诉他'从这儿送到那儿'，不用管他走哪条路。你要**信任递归**！只要出口对、递归公式对，递归一定能帮你搞定！"

> - **队列**就像食堂排队：先来的先打饭（先进先出）
>
> 有意思的是，递归的背后其实就藏着一个"栈"！下节课你们就会明白，为什么递归忘写出口会"栈溢出"了。
