## 四、讲稿提纲

### 4.1 导入回顾（8分钟）——"手洗衣服太累了！"

> 同学们，上两节课我们学了三种排序：冒泡排序、选择排序、插入排序。大家还记得吗？写冒泡排序的时候，要写两层for循环，还要比较、交换，代码差不多要十几行，写起来是不是有点累？
>
> 老师给大家打个比方——之前我们手写排序，就好比**用手洗衣服**。你要一件一件搓，搓完还要拧，拧完还要晾。能洗干净吗？能！但是累不累？特别累！
>
> 那现实生活中，大家家里洗衣服用什么？——**洗衣机**！把衣服丢进去，按一下按钮，它自己就洗好了，又快又干净。
>
> 今天，老师就要教大家C++里的"洗衣机"——一个叫 `sort` 的函数。以后排序再也不用自己写那么多代码了，一行代码搞定！

**（在黑板上写：手写排序 = 手洗衣服，sort = 洗衣机）**

### 4.2 新课讲解上半场（20分钟）——认识STL和sort

#### 第一步：STL是什么？（5分钟）

> 在讲sort之前，老师先告诉大家一个概念：**STL**。
>
> STL是三个英文单词的缩写，全名你们不用记，只要知道它是什么就行——
>
> **STL就是C++自带的一个大工具箱。**
>
> 想象一下，你买了一套乐高积木，盒子里除了积木块，还送了一把小起子、一个小钳子对吧？这些工具是别人帮你做好的，你直接拿来用就行。
>
> STL就是C++这门语言"盒子里附送的工具"。这些工具都是世界上最厉害的程序员帮我们写好的，我们直接用就行，不用自己造。
>
> **sort就是STL工具箱里的一把"排序神器"。**

#### 第二步：请出sort——algorithm头文件（3分钟）

> 要用sort，我们得先告诉电脑："嘿，我要用工具箱里的东西了！"怎么告诉它呢？在代码最上面加一行：

```cpp
#include<algorithm>
```

> `algorithm` 是一个英文单词，意思是"算法"。你可以理解为：这一行是在说"我要打开算法工具箱"。
>
> 以后凡是要用sort，别忘了加这行。就像去游泳要带泳衣一样，用sort就要带上algorithm！

**（板书小口诀：sort排序，带algorithm）**

#### 第三步：sort怎么用？（12分钟）

> 好，现在来看sort怎么用。假设我们有一个数组：

```cpp
int a[5] = {5, 3, 1, 4, 2};
```

> 我们想把它从小到大排好。用sort只需要**一行代码**：

```cpp
sort(a, a + 5);
```

> 排完之后，数组就变成了 `{1, 2, 3, 4, 5}`。是不是超级简单？
>
> 来，我们仔细看看这一行代码：
> - `sort` 是函数名，就是"排序"的意思
> - 括号里有两个东西，用逗号隔开
> - 第一个 `a` ——告诉sort"从哪里开始排"，就是数组的第一个元素
> - 第二个 `a + 5` ——告诉sort"排到哪里为止"，5就是数组里有几个元素
>
> 打个比方：你跟妈妈说"帮我把书架**从第1格到第5格**整理一下"，sort就需要知道"从哪到哪"。
>
> **如果数组有n个元素，就写 `sort(a, a + n)`。**记住这个公式就行了！

**（板书公式框：`sort(a, a + n);`  ← 对a数组的前n个元素从小到大排序）**

> 默认情况下，sort帮我们**从小到大**排。就像你跟洗衣机说"普通洗"，它就按默认模式来。

### 4.3 新课讲解下半场（15分钟）——从大到小 + sort vs 手写

#### 第四步：从大到小排序（8分钟）

> 那如果我想从大到小排呢？比如考试成绩，我想看谁考得最高排在前面。
>
> 这时候要多加一个东西，告诉sort"你反着排"：

```cpp
sort(a, a + n, greater<int>());
```

> 第三个参数 `greater<int>()`，你不用理解它是什么意思，就把它**当成一个咒语来记**：
> - `greater` 是"更大"的意思
> - 尖括号里放 `int`（因为我们排的是整数）
> - 最后有一对小圆括号 `()`，千万别忘了！
>
> 老师教你一个记忆口诀：**"greater尖int圆"**——greater后面跟尖括号放int，再跟圆括号。

**（板书：从大到小咒语 → `greater<int>()`  口诀：greater尖int圆）**

> 来看完整的例子：

```cpp
int a[5] = {5, 3, 1, 4, 2};
sort(a, a + 5, greater<int>());
// 排完变成：{5, 4, 3, 2, 1}
```

#### 第五步：sort vs 手写排序（7分钟）

> 现在我们来对比一下。

**（板书两列对比表）**

> | | 手写排序（手洗衣服） | sort（洗衣机） |
> |---|---|---|
> | 代码量 | 十几行 | 一行 |
> | 速度 | 慢（冒泡排序特别慢） | 非常快 |
> | 出错率 | 容易写错 | 不容易错 |
> | 竞赛建议 | 了解原理就行 | **优先使用** |
>
> 在CSP-J比赛中，时间很宝贵，能一行搞定的事，我们就不写十几行。所以**竞赛中排序优先用sort**，除非题目特别要求你手写。
>
> 但是！上两节课学的排序原理不是白学的哦。了解原理才能更好地使用工具。就像你知道洗衣机的原理是"搅动+水流冲洗"，你才知道什么衣服能丢进去洗、什么不能。

## 五、课堂例题

### 例题1：小小排序（从小到大）

**题目：** 输入n个整数，从小到大排序后输出。

**输入样例：**
```
5
5 3 1 4 2
```

**输出样例：**
```
1 2 3 4 5
```

**完整代码：**

```cpp
#include<iostream>
#include<algorithm>
using namespace std;
int main(){
    int n;
    cin >> n;
    int a[1005];
    for(int i = 0; i < n; i++){
        cin >> a[i];
    }
    sort(a, a + n);  // 一行搞定排序！
    for(int i = 0; i < n; i++){
        if(i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;
    return 0;
}
```

**讲解思路：**

> 大家看，整个代码的核心就是中间那一行 `sort(a, a + n);`。输入n个数，调用sort，输出，完事了！
>
> 跟上两节课的冒泡排序比，我们少写了两层for循环和交换的代码，是不是轻松多了？
>
> 注意两个地方：
> 1. 别忘了 `#include<algorithm>`，不写的话编译会报错。
> 2. `sort(a, a + n)` 里的n就是数组元素个数，别写成 `a + n - 1`，那样最后一个元素就没排上。

---

### 例题2：期末成绩单（从大到小）

**题目：** 老师录入了全班n位同学的数学成绩，请帮老师按分数**从高到低**排好，输出排序后的成绩。

**输入样例：**
```
6
88 95 72 100 65 95
```

**输出样例：**
```
100 95 95 88 72 65
```

**完整代码：**

```cpp
#include<iostream>
#include<algorithm>
using namespace std;
int main(){
    int n;
    cin >> n;
    int a[1005];
    for(int i = 0; i < n; i++){
        cin >> a[i];
    }
    sort(a, a + n, greater<int>());  // 从大到小
    for(int i = 0; i < n; i++){
        if(i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;
    return 0;
}
```

**讲解思路：**

> 跟例题1几乎一模一样，唯一的区别就是sort那一行多了第三个参数 `greater<int>()`。
>
> 同学们试着自己写一遍，特别注意 `greater<int>()` 的写法：
> - `greater` 后面是**尖括号** `<int>`，不是圆括号
> - 最后还有一对**圆括号** `()`，不能少
>
> 记住口诀："greater尖int圆"！

---

### 例题3：谁是前三名？

**题目：** 运动会跳远比赛，n位同学参赛，每人跳一次，记录成绩（单位：厘米）。请找出前三名的成绩并输出（如果不足3人，有几个输出几个）。

**输入样例：**
```
5
320 285 350 310 340
```

**输出样例：**
```
350 340 320
```

**完整代码：**

```cpp
#include<iostream>
#include<algorithm>
using namespace std;
int main(){
    int n;
    cin >> n;
    int a[1005];
    for(int i = 0; i < n; i++){
        cin >> a[i];
    }
    sort(a, a + n, greater<int>());  // 从大到小排
    int top = 3;
    if(n < 3) top = n;  // 不足3人的情况
    for(int i = 0; i < top; i++){
        if(i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;
    return 0;
}
```

**讲解思路：**

> 这道题比前两道多了一步——排好序之后，我们只需要输出前3个。
>
> 关键思路：先用sort从大到小排，排完之后，a[0]就是最大的，a[1]是第二大的，a[2]是第三大的。直接输出前三个就行。
>
> 但要注意一个特殊情况：万一只有2个人或1个人参赛呢？所以加了一个判断 `if(n < 3) top = n;`。
>
> 这道题告诉我们：**sort不仅是用来排序输出的，排完序之后还可以方便地找最大、最小、前几名。** 后面学贪心算法的时候，sort会是我们最好的帮手！

## 七、常见错误与应对

### 错误1：忘记写 `#include<algorithm>`

```
error: 'sort' was not declared in this scope
```

**应对：** 看到这个报错，第一反应检查有没有写 `#include<algorithm>`。口诀：**"用sort，带algorithm"**。可以建议学生养成习惯——开头直接把 `<iostream>` 和 `<algorithm>` 都写上。

---

### 错误2：`sort(a, a+n-1)` 少排了一个元素

**现象：** 数组最后一个元素没参与排序。

**应对：** 强调 `a+n` 是"排到第n个元素为止"，不需要减1。类比：你说"从第1页看到第5页"，是看5页，不是4页。sort的第二个参数是"终点的下一个位置"。

---

### 错误3：`greater<int>` 后面忘写 `()`

```cpp
sort(a, a+n, greater<int>);   // 错误！少了()
sort(a, a+n, greater<int>()); // 正确！
```

**应对：** 反复提醒口诀 **"greater尖int圆"**，尤其圆括号不能少。可以让学生在草稿纸上写三遍加深印象。

---

### 错误4：`greater(int)` 括号类型写反

```cpp
sort(a, a+n, greater(int));     // 错误！
sort(a, a+n, greater<int>());   // 正确！
```

**应对：** 在黑板上把正确写法用彩色粉笔标注：`greater`**<**`int`**>**`()`，让学生看清楚哪个是尖括号、哪个是圆括号。

---

### 错误5：数组开太小导致越界

```cpp
int a[5];  // 只开了5个，但输入了100个数
```

**应对：** 提醒学生竞赛习惯——数组大小**多开一些**，一般题目数据范围最大是多少，就开多少+5。推荐统一开 `a[100005]`。

## 九、下节课预告

> 同学们，今天我们学会了sort这把"排序神器"，一行代码就能搞定排序。下节课，我们要学一个特别有意思的算法思想——**贪心算法**。
>
> 什么是贪心呢？就像你在自助餐厅吃饭，每次都挑你最想吃的菜。贪心算法就是"每一步都选当前最好的"。
>
> 而且告诉大家一个小秘密：贪心算法的很多题目，**第一步就是sort排序**！所以今天学的内容，下节课马上就能用上。
>
> 回家记得完成洛谷上的两道作业题，下节课见！
