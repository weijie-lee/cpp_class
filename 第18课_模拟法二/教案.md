>
> 问题来了：**谁是最后的幸运之星？**

### 4.2 报数淘汰的思路分析（讲解①）

**核心比喻——"站着"和"蹲下"：**

> 想象 8 个小朋友站成一排（我们用数组表示）。
> - 站着的 = 还没被淘汰（标记为 1）
> - 蹲下的 = 已经被淘汰了（标记为 0）
>
> 我们从头到尾一个一个数。数到蹲下的人就跳过（人家已经出局了嘛），只数站着的人。
> 数到 3 的人——"砰！请蹲下！"
>
> 数到最后一个人后怎么办？**绕回第一个人**继续数！就像圆圈一样。

**板书演示过程：**

```
初始状态：  [站][站][站][站][站][站][站][站]
编号：        0    1    2    3    4    5    6    7

第 1 轮报数：
  0号报1，1号报2，2号报3 → 2号蹲下！
  [站][站][蹲][站][站][站][站][站]

第 2 轮报数：
  3号报1，4号报2，5号报3 → 5号蹲下！
  [站][站][蹲][站][站][蹲][站][站]

……依此类推，直到只剩一个人站着。
```

**编程思路（中文版，先不急写代码）：**

1. 造一个数组 `alive[8]`，全部填 1（表示都站着）
2. 用一个变量 `cnt` 记录"当前报到几了"
3. 用一个变量 `left` 记录"还剩几个人站着"
4. 从第 0 个人开始，一个一个看：
   - 如果他是站着的（`alive[i] == 1`），`cnt` 加 1
   - 如果 `cnt` 数到了 3，这个人蹲下（`alive[i] = 0`），`left` 减 1，`cnt` 归零
5. 数到第 7 个人之后，**绕回第 0 个人**（取余：`i = (i + 1) % n`）
6. 当 `left == 1` 时，找到那个还站着的人，输出他的编号

> 🔑 **关键概念——取余实现"绕回"：**
>
> 同学们还记得取余运算 `%` 吗？
> - `7 % 8 = 7`（还没到头）
> - `8 % 8 = 0`（到头了，绕回第 0 个！）
>
> 就像操场上跑圈，跑到终点又回到起点。`%` 就是帮我们"自动绕回"的魔法符号！

### 4.3 凯撒密码的故事（讲解②）

> 同学们，你们想不想学一门**密码术**？
>
> 两千多年前，古罗马有个大将军叫**凯撒**（就是"凯撒大帝"）。他在打仗时需要给远方的将领写信，但又怕信被敌人截获看懂。于是他想了一个办法——
>
> **把信里每个字母往后移 3 位！**
> - a → d
> - b → e
> - c → f
> - ……
> - x → a（超过 z 就绕回来！）
> - y → b
> - z → c
>
> 这样敌人拿到信，看到的全是乱码！只有知道"往回移 3 位"秘密的人才能读懂。
>
> 这就是著名的**凯撒密码**！今天我们来用 C++ 写一个加密程序。

**思路分析：**

```
原文：  h e l l o
偏移3： k h o o r

规则：新字母 = (原字母 - 'a' + 3) % 26 + 'a'
```

> 为什么要先减 'a'？因为我们要把字母变成 0~25 的数字来算。
> - 'a' - 'a' = 0
> - 'b' - 'a' = 1
> - 'z' - 'a' = 25
>
> 加上偏移量 3 之后，如果超过 25 了怎么办？**取余 26**，就绕回来了！
>
> 最后再加上 'a'，把数字变回字母。
>
> 看，又是"绕回"！和刚才的报数游戏一模一样的思路。

### 4.4 调试技巧——打印中间变量（讲解③）

> 同学们，写模拟题的时候，代码经常很长，一出错就不知道错在哪里。
>
> 老师教你们一个"找 Bug 神器"——**打印中间变量！**
>
> 就好比你做数学题，不会一步算出最后的答案，你会在草稿纸上写出每一步的中间结果，然后检查哪一步算错了。
>
> 写程序也一样！在关键步骤后面加一句 `cout`，把变量的值打印出来看看。

**演示方法：**

```cpp
// 调试版本（检查报数过程）
if (alive[i] == 1) {
    cnt++;
    cout << "DEBUG: 编号" << i << "报数" << cnt << endl;  // ← 调试输出
    if (cnt == m) {
        alive[i] = 0;
        left--;
        cnt = 0;
        cout << "DEBUG: 编号" << i << "被淘汰，剩余" << left << "人" << endl;  // ← 调试输出
    }
}
```

> 等程序跑对了，再把这些 `cout` 删掉或者注释掉就好啦。
>
> 🎯 **记住这句话：模拟题出了 Bug，先打印中间变量，一步一步检查！**

## 五、课堂例题

### 例题 1：报数淘汰——约瑟夫环简化版（教师演示 + 学生跟敲）

**题目描述：**
n 个小朋友（编号 1 到 n）围成一圈，从 1 号开始报数，每次报到 m 的人出列。输出最后剩下的那个人的编号。

**输入：** 一行两个整数 n 和 m（n ≤ 100，m ≤ 10）
**输出：** 最后剩下的人的编号

**讲解思路：**
1. 用数组 `alive[i]` 表示第 i 个人是否还在（1=在，0=淘汰）。
2. 用变量 `cnt` 记当前报了几个数，`left` 记还剩几个人。
3. 用 `pos` 表示当前走到了哪个位置，每次 `pos = (pos + 1) % n` 实现绕回。
4. 当 `left == 1` 时停止，找到那个 `alive[i] == 1` 的人。

```cpp
#include <iostream>
using namespace std;

int alive[105];

int main() {
    int n, m;
    cin >> n >> m;

    // 初始化：所有人都站着
    for (int i = 0; i < n; i++) {
        alive[i] = 1;
    }

    int left = n;   // 剩余人数
    int cnt = 0;    // 当前报数计数
    int pos = 0;    // 当前位置（从0号开始）

    while (left > 1) {
        if (alive[pos] == 1) {  // 这个人还在
            cnt++;
            if (cnt == m) {     // 报到 m 了，淘汰！
                alive[pos] = 0;
                left--;
                cnt = 0;        // 重新开始计数
            }
        }
        pos = (pos + 1) % n;   // 移到下一个位置（绕回）
    }

    // 找到最后一个还站着的人
    for (int i = 0; i < n; i++) {
        if (alive[i] == 1) {
            cout << i + 1 << endl;  // 编号从1开始，所以加1
            break;
        }
    }

    return 0;
}
```

**运行示例：**
```
输入：8 3
输出：7
```

**验证过程（板书画）：**
```
8人（编号1~8，下标0~7），报到3淘汰：
第1个淘汰：3号（下标2）
第2个淘汰：6号（下标5）
第3个淘汰：1号（下标0）
第4个淘汰：5号（下标4）
第5个淘汰：2号（下标1）
第6个淘汰：8号（下标7）
第7个淘汰：4号（下标3）
最后剩下：7号（下标6）✓
```

**教师讲解要点：**
- 重点讲 `pos = (pos + 1) % n` 的作用——自动绕回
- 提问："`% n` 去掉会怎样？"→ pos 会一直增大，访问数组越界！
- 提问："为什么输出 `i + 1` 而不是 `i`？"→ 题目编号从 1 开始，数组下标从 0 开始
- 带学生手动走一遍前 3 轮，和程序输出对照

### 例题 2：凯撒密码加密（学生跟练）

**题目描述：**
输入一个只含小写字母的字符串 s 和一个整数 k（1 ≤ k ≤ 25），将字符串中每个字母向后移动 k 位（超过 z 绕回 a），输出加密后的字符串。

**输入：** 第一行一个字符串（长度 ≤ 100），第二行一个整数 k
**输出：** 加密后的字符串

**讲解思路：**
1. 用字符数组（或逐字符读取）存原文。
2. 对每个字母：先减 'a' 变成 0~25 的数字，加上 k，取余 26，再加 'a' 变回字母。
3. 逐个输出。

```cpp
#include <iostream>
#include <cstring>
using namespace std;

char s[105];

int main() {
    int k;
    cin >> s >> k;

    int len = strlen(s);  // 字符串长度

    for (int i = 0; i < len; i++) {
        // 第1步：字母变数字（a=0, b=1, ..., z=25）
        int num = s[i] - 'a';
        // 第2步：加上偏移量，取余绕回
        int newNum = (num + k) % 26;
        // 第3步：数字变回字母
        char newChar = newNum + 'a';
        cout << newChar;
    }
    cout << endl;

    return 0;
}
```

**运行示例：**
```
输入：
hello
3
输出：
khoor
```

**再试一个：**
```
输入：
xyz
3
输出：
abc
```

**教师讲解要点：**
- 分三步讲：字母→数字→加密计算→数字→字母，每步单独理解
- 重点讲 `% 26` 的作用：x(23)+3=26，26%26=0，0+'a'='a'，完美绕回！
- 让学生手动算 'y'+3：y=24，24+3=27，27%26=1，1+'a'='b' ✓
- 提问："如果不写 `% 26`，加密 'z' 会怎样？"→ 25+3=28，28+'a' 不是字母了，乱码！
- 🎉 **成就感设计**："你们刚刚学会了凯撒大帝的密码术！回家可以用这个方法和好朋友写秘密信哦！"

### 例题 3：用打印调试修复 Bug（教师演示）

**题目描述：**
老师故意写了一段"有 Bug 的报数淘汰程序"，和学生一起用"打印中间变量"的方法找出 Bug。

**有 Bug 的代码：**

```cpp
#include <iostream>
using namespace std;

int alive[105];

int main() {
    int n, m;
    cin >> n >> m;

    for (int i = 0; i < n; i++) {
        alive[i] = 1;
    }

    int left = n;
    int cnt = 0;
    int pos = 0;

    while (left > 1) {
        if (alive[pos] == 1) {
            cnt++;
            if (cnt == m) {
                alive[pos] = 0;
                left--;
                // Bug在这里：忘记把 cnt 归零！
            }
        }
        pos = (pos + 1) % n;
    }

    for (int i = 0; i < n; i++) {
        if (alive[i] == 1) {
            cout << i + 1 << endl;
            break;
        }
    }

    return 0;
}
```

**调试过程演示：**

> 老师："这段代码输入 `5 2` 应该输出 `3`，但实际跑出来不对。我们怎么办？"
>
> "来，我们在关键位置加上 `cout`，看看每一步发生了什么——"

```cpp
if (alive[pos] == 1) {
    cnt++;
    cout << "DEBUG: pos=" << pos << " cnt=" << cnt << endl;  // 加上这句！
    if (cnt == m) {
        alive[pos] = 0;
        left--;
        cout << "DEBUG: 淘汰pos=" << pos << " left=" << left << endl;  // 加上这句！
    }
}
```

> 运行后发现 `cnt` 一直在增长，报到 2 之后没有归零，变成了 3、4、5……
>
> "找到了！`cnt` 忘记归零了！加上 `cnt = 0;` 就好了。"
>
> "看，打印中间变量就像给程序做体检，哪里不正常一眼就看出来了！"

**教师讲解要点：**
- 不要直接告诉答案，先让学生观察调试输出，引导他们自己发现 `cnt` 没有归零
- 强调：写模拟题的时候，一定要有意识地"检查每一步"
- 修好 Bug 之后，提醒学生删掉或注释掉调试输出

## 六、课堂练习题

### 练习 1：反向凯撒——解密（⭐⭐）

**题目：**
凯撒密码的解密是加密的逆过程：把每个字母**往前移** k 位。输入一个只含小写字母的密文字符串和整数 k（1 ≤ k ≤ 25），输出解密后的原文。

**样例输入：**
```
khoor
3
```
**样例输出：**
```
hello
```

**提示：** 往前移 k 位 = 往后移 (26 - k) 位。所以公式变成 `(num + 26 - k) % 26 + 'a'`。为什么要加 26？因为如果 `num - k` 是负数，取余会出问题，加 26 保证结果是正数。

**参考答案：**
```cpp
#include <iostream>
#include <cstring>
using namespace std;

char s[105];

int main() {
    int k;
    cin >> s >> k;

    int len = strlen(s);
    for (int i = 0; i < len; i++) {
        int num = s[i] - 'a';
        int newNum = (num + 26 - k) % 26;  // 往前移 k 位
        cout << (char)(newNum + 'a');
    }
    cout << endl;

    return 0;
}
```

### 练习 2：报数淘汰——输出淘汰顺序（⭐⭐）

**题目：**
n 个人（编号 1~n）围成一圈，从 1 号开始报数，报到 m 的人出列。请**按淘汰顺序**输出每个人的编号。

**输入：** 一行两个整数 n 和 m
**输出：** 按淘汰顺序输出 n 个编号，空格隔开

**样例输入：**
```
5 2
```
**样例输出：**
```
2 4 1 5 3
```

**提示：** 在例题 1 的基础上，每次有人淘汰时就立刻输出他的编号，循环条件改成 `left > 0`（所有人都要输出）。

**参考答案：**
```cpp
#include <iostream>
using namespace std;

int alive[105];

int main() {
    int n, m;
    cin >> n >> m;

    for (int i = 0; i < n; i++) {
        alive[i] = 1;
    }

    int left = n, cnt = 0, pos = 0;

    while (left > 0) {
        if (alive[pos] == 1) {
            cnt++;
            if (cnt == m) {
                alive[pos] = 0;
                left--;
                cnt = 0;
                cout << pos + 1 << " ";  // 淘汰时立刻输出
            }
        }
        pos = (pos + 1) % n;
    }
    cout << endl;

    return 0;
}
```

### 练习 3：数字加密器（⭐⭐⭐）

**题目：**
小明想给一个四位数加密。加密规则如下：
1. 将每一位数字都加上 5；
2. 用加上 5 后的数除以 10 的余数代替该位数字；
3. 将第 1 位和第 3 位交换，第 2 位和第 4 位交换。

输入一个四位正整数，输出加密后的四位数（若高位为 0 也要输出）。

**样例输入：**
```
1234
```
**样例输出：**
```
8967
```

**解释：** 1234 → 每位加5 → 6789 → 取余10 → 6789 → 交换 → 8967

**再试一个：**
```
输入：9876
加5后：14 13 12 11 → 取余10 → 4321 → 交换 → 2143
输出：2143
```

**参考答案：**
```cpp
#include <iostream>
using namespace std;

int main() {
    int x;
    cin >> x;

    int a[4];
    // 拆位（从低位到高位）
    a[3] = x % 10; x /= 10;
    a[2] = x % 10; x /= 10;
    a[1] = x % 10; x /= 10;
    a[0] = x % 10;

    // 每位加 5 再取余 10
    for (int i = 0; i < 4; i++) {
        a[i] = (a[i] + 5) % 10;
    }

    // 交换第1位和第3位（下标0和2）
    int t = a[0]; a[0] = a[2]; a[2] = t;
    // 交换第2位和第4位（下标1和3）
    t = a[1]; a[1] = a[3]; a[3] = t;

    // 输出
    for (int i = 0; i < 4; i++) {
        cout << a[i];
    }
    cout << endl;

    return 0;
}
```

## 七、常见错误与应对

| # | 常见错误 | 错误代码示例 | 正确写法 | 应对方法 |
|---|---------|------------|---------|---------|
| 1 | **忘记用取余实现绕回** | `pos = pos + 1;`（数到最后一个人就不回来了） | `pos = (pos + 1) % n;` | 用"跑圈"比喻：跑到终点不停下来，继续从起点跑 |
| 2 | **报数后忘记将计数器归零** | 淘汰一个人后 `cnt` 继续累加 | 淘汰后加 `cnt = 0;` | 用调试输出打印 `cnt`，让学生自己发现"cnt 怎么越来越大" |
| 3 | **凯撒密码忘记取余 26** | `newChar = (s[i] - 'a' + k) + 'a';` | `newChar = (s[i] - 'a' + k) % 26 + 'a';` | 让学生试试加密 'z'，没有 `% 26` 会得到乱码 |
| 4 | **解密时直接减 k 导致负数** | `(num - k) % 26`（当 num < k 时结果为负） | `(num + 26 - k) % 26` | 讲解"先加 26 再减，保证不会变负数"的技巧 |
| 5 | **编号差 1（off-by-one）** | 输出 `i` 而不是 `i + 1` | 数组下标从 0 开始，题目编号从 1 开始，输出时 `+1` | 每次都提醒：题目编号和数组下标差 1 |
| 6 | **跳过已淘汰的人时逻辑出错** | 在 `alive[pos]==0` 时也执行了 `cnt++` | `cnt++` 必须放在 `alive[pos]==1` 的条件里 | 强调"蹲下的人不报数"，判断条件一定要先检查 |

## 九、下节课预告

> 🔮 **第 19 课：冒泡排序**
>
> 同学们，今天我们学会了用数组模拟各种复杂的过程——报数淘汰、凯撒密码……你们已经很厉害了！
>
> 下节课，我们要解决一个经典问题：**怎么把一堆乱七八糟的数字从小到大排好？**
>
> 想象一下水里的气泡——大泡泡会"咕噜咕噜"往上浮，小泡泡留在下面。排序也是这样，大的数一点一点"浮"到后面去。这就是大名鼎鼎的**冒泡排序**！
>
> 它是我们学的第一个"排序算法"，也是信息学竞赛中最基础的算法之一。回家把今天的两道作业做完，下节课见！

> 📝 **备课备注（仅教师可见）：**
> - 板书准备：提前画好 8 个人围成圈的示意图；准备凯撒密码的对照表（a→d, b→e, ... z→c）。
> - 35′-40′ 的"真人报数淘汰"游戏：让 8 个同学站起来围成圈，真的报数，报到 3 的坐下。这个活动既巩固理解又管理注意力，强烈建议执行。
> - 凯撒密码部分只处理小写字母，不涉及大小写混合和非字母字符，降低难度。如果有学生问到大写字母，可以口头说"思路一样，只是把 'a' 换成 'A'"，不展开。
> - `strlen()` 函数第一次出现，只需告诉学生"它能算出字符串有几个字符"，不深入讲 `<cstring>` 头文件的细节。
> - 调试技巧的演示非常重要，这是学生第一次系统接触"怎么找 Bug"。建议故意"演"得夸张一点："哎呀出错了，怎么办？别慌，我们来当侦探！"增加趣味性。
> - 对提前完成的同学，可以口头附加题："能不能同时支持大小写字母的凯撒密码？"或"如果偏移量大于 26 怎么办？"
