## 四、讲稿提纲

### 4.1 故事导入："自助餐大作战"（5 分钟）

> 同学们，假设你去吃自助餐🍽️，肚子只能装下 5 盘菜。菜台上有很多菜：炸鸡 38 元、三文鱼 58 元、白米饭 3 元、炒青菜 8 元、大龙虾 88 元、牛排 68 元、蒸蛋 5 元……
>
> 你会怎么选？是不是先挑最贵的？大龙虾！牛排！三文鱼！炸鸡！然后再来一个……对吧？
>
> 这就是**贪心**的思想——**每一步都选当前看起来最好的**。你不需要把所有组合都试一遍（那是枚举法），你只要**每次拿最好的**就行了。

**板书/PPT 关键词：**
```
贪心 = 每一步选最好的 → 局部最优 → 整体最优
```

### 4.2 讲解一：找零钱问题（15 分钟）

#### 引入

> 老师去超市买东西，要找给顾客 **36 元**零钱。收银台里有四种面额的硬币：**10 元、5 元、2 元、1 元**，每种都有很多个。
>
> 问题：**怎样用最少的硬币凑出 36 元？**

#### 带着学生思考

> 如果用枚举法，我们要试所有组合……太累了！
>
> 换个思路：你是收银员，你肯定想**先用大的**对吧？
>
> - 先看 10 元：36 ÷ 10 = 3 余 6，用掉 3 个 10 元，还剩 6 元
> - 再看 5 元：6 ÷ 5 = 1 余 1，用掉 1 个 5 元，还剩 1 元
> - 再看 2 元：1 ÷ 2 = 0 余 1，用不了
> - 最后 1 元：1 ÷ 1 = 1，用掉 1 个 1 元
>
> 总共：3 + 1 + 0 + 1 = **5 个硬币**。这就是最少的！

#### 归纳贪心策略

> **策略：每次都尽量用面额最大的硬币。**
>
> 为什么这样是对的？因为一个 10 元硬币相当于两个 5 元、五个 2 元或十个 1 元。用一个大的，肯定比用好几个小的"划算"。

#### 提醒（埋伏笔）

> ⚠️ 但是注意：不是所有零钱问题都能贪心！
>
> 比如硬币面额是 1、3、4 元，要凑 6 元。贪心会选：4+1+1 = 3 个硬币，但其实 3+3 = 2 个硬币更少！
>
> 所以贪心有个前提：**你要确认"每次选最大的"确实是对的**。人民币的面额刚好满足这个条件，但不是所有情况都行哦。

### 4.3 小游戏："能不能贪心？"抢答（5 分钟）

老师说一个场景，学生判断能不能用贪心：

1. ✅ 超市找零（人民币面额 100、50、20、10、5、1）→ **能！**
2. ❌ 硬币面额 1、3、4 凑 6 元 → **不能！**（贪心得到 3 枚，最优是 2 枚）
3. ✅ 装糖果到口袋，口袋有容量限制，糖果大小不同，先装最小的 → **能！**（装得最多）
4. ✅ 排队打饭，让做菜最快的人先打，大家总等待时间最少 → **能！**

> 记住口诀：**贪心不是万能的，但在对的场景里，它又快又好！**

### 4.4 讲解二：活动安排问题（20 分钟）

#### 引入

> 周末你有一天自由时间，有好多个兴趣班可以去试听。每个兴趣班的**开始时间和结束时间**不一样。你同一个时间只能待在一个地方。
>
> **问题：你最多能参加几个兴趣班？**

#### 举例

| 兴趣班 | 开始时间 | 结束时间 |
|--------|---------|---------|
| 编程课 | 8:00 | 10:00 |
| 画画课 | 9:00 | 11:00 |
| 围棋课 | 10:00 | 12:00 |
| 书法课 | 11:00 | 13:00 |
| 舞蹈课 | 8:00 | 9:00 |

#### 带学生推理

> 如果用枚举法，5 个活动的所有组合有 2⁵ = 32 种，还得判断有没有时间冲突……太麻烦了！
>
> 贪心策略是什么呢？想一想：**你应该优先选那些"结束得早"的活动！**
>
> 为什么？因为一个活动越早结束，你就越早腾出时间去参加下一个活动，这样能塞进去的活动就越多。
>
> 就像吃自助餐先挑小份甜点尝味道，省出肚子吃更多花样！

#### 手动模拟

> 1. 先按**结束时间**从小到大排序：
>    - 舞蹈课 8:00–9:00
>    - 编程课 8:00–10:00
>    - 画画课 9:00–11:00
>    - 围棋课 10:00–12:00
>    - 书法课 11:00–13:00
>
> 2. 选第一个：**舞蹈课**（结束时间 9:00）✅
> 3. 编程课 8:00 开始，但 8:00 < 9:00，和舞蹈课冲突 ❌
> 4. 画画课 9:00 开始，9:00 ≥ 9:00，不冲突 ✅
> 5. 围棋课 10:00 开始，10:00 < 11:00，和画画课冲突 ❌
> 6. 书法课 11:00 开始，11:00 ≥ 11:00，不冲突 ✅
>
> 答案：最多参加 **3** 个（舞蹈、画画、书法）。

### 4.5 贪心 vs 枚举（5 分钟）

| | 枚举法 | 贪心算法 |
|---|-------|---------|
| **思路** | 把所有可能都试一遍，选最好的 | 每一步直接选当前最好的 |
| **速度** | 通常很慢（组合数爆炸） | 通常很快（排个序就搞定） |
| **正确性** | 一定能找到最优解 | 只在"合适的问题"上能找到最优解 |
| **比喻** | 考试时每道题都验算三遍 | 学霸直觉秒答，还都答对了 |
| **适用场景** | 数据小、没有好策略时 | 能证明"每步选最好=全局最好"时 |

> 枚举是"笨办法但一定对"，贪心是"聪明办法但要看场景"。

## 五、课堂例题

### 例题 1：找零钱（最少硬币）

**题目描述：**

有四种硬币，面额分别为 10 元、5 元、2 元、1 元，每种数量无限。输入一个正整数 n（1 ≤ n ≤ 1000），输出凑出 n 元最少需要多少个硬币，以及每种硬币用了多少个。

**输入样例：**
```
36
```

**输出样例：**
```
10元硬币:3个
5元硬币:1个
2元硬币:0个
1元硬币:1个
最少需要5个硬币
```

**讲解思路：**

1. 把面额从大到小排好：10、5、2、1。
2. 从最大面额开始，尽量多用：`个数 = 剩余金额 / 面额`，然后 `剩余金额 = 剩余金额 % 面额`。
3. 循环处理每个面额，最后输出结果。

**完整代码：**

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    int coins[] = {10, 5, 2, 1};  // 面额从大到小
    int count[4] = {0};           // 记录每种硬币用了几个
    int total = 0;                // 总硬币数
    int remain = n;               // 剩余金额

    for (int i = 0; i < 4; i++) {
        count[i] = remain / coins[i];  // 尽量多用当前面额
        remain = remain % coins[i];    // 更新剩余金额
        total = total + count[i];      // 累加硬币数
    }

    // 输出结果
    for (int i = 0; i < 4; i++) {
        cout << coins[i] << "元硬币:" << count[i] << "个" << endl;
    }
    cout << "最少需要" << total << "个硬币" << endl;

    return 0;
}
```

**代码逐行讲解：**

- `int coins[] = {10, 5, 2, 1};` → 面额数组，**从大到小**排列，这是贪心的关键！
- `count[i] = remain / coins[i];` → 整除，算出当前面额最多能用几个。
- `remain = remain % coins[i];` → 取余，算出剩下还需要凑多少。
- 整个循环就是"先用大的，用不了再用小的"——这就是贪心！

### 例题 2：活动安排（简化版）

**题目描述：**

小明周末想参加尽可能多的兴趣班。有 n 个兴趣班（1 ≤ n ≤ 100），每个兴趣班有开始时间和结束时间（用整数表示）。同一时间只能参加一个兴趣班。如果一个兴趣班在时间 t 结束，另一个在时间 t 开始，可以都参加（来得及赶过去）。

输入第一行为 n，接下来 n 行每行两个整数表示开始时间和结束时间。输出最多能参加几个兴趣班。

**输入样例：**
```
5
8 10
9 11
10 12
11 13
8 9
```

**输出样例：**
```
3
```

**讲解思路：**

1. **关键策略：按结束时间从小到大排序。**
2. 选第一个活动（结束最早的）。
3. 从第二个活动开始，如果它的开始时间 ≥ 上一个选中活动的结束时间，就选它。
4. 统计选了几个。

> 🎯 为什么按结束时间排？因为越早结束，留给后面的时间越多，能安排的活动就越多。

**完整代码：**

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

struct Activity {
    int start;  // 开始时间
    int end;    // 结束时间
};

// 比较函数：按结束时间从小到大排序
bool cmp(Activity a, Activity b) {
    return a.end < b.end;
}

int main() {
    int n;
    cin >> n;

    Activity a[105];
    for (int i = 0; i < n; i++) {
        cin >> a[i].start >> a[i].end;
    }

    // 第一步：按结束时间排序（贪心的预处理）
    sort(a, a + n, cmp);

    // 第二步：贪心选择
    int count = 1;           // 第一个活动一定选
    int lastEnd = a[0].end;  // 记录上一个选中活动的结束时间

    for (int i = 1; i < n; i++) {
        if (a[i].start >= lastEnd) {  // 不冲突，可以选
            count++;
            lastEnd = a[i].end;       // 更新结束时间
        }
    }

    cout << count << endl;

    return 0;
}
```

**代码逐行讲解：**

- `struct Activity` → 结构体，把开始时间和结束时间"打包"在一起。（如果学生还没学结构体，可以用两个数组 `s[]` 和 `e[]` 代替，见下方备用版本。）
- `bool cmp(...)` → 自定义比较函数，告诉 `sort` 按结束时间排序。
- `sort(a, a + n, cmp);` → 上节课学的 `sort`，这里派上大用场了！
- `if (a[i].start >= lastEnd)` → 判断新活动和上一个选中的活动有没有时间冲突。

**备用版本（不用结构体，用两个数组）：**

```cpp
#include <iostream>
#include <algorithm>
using namespace std;

int s[105], e[105], order[105];  // 开始时间、结束时间、排序用的下标

bool cmp(int i, int j) {
    return e[i] < e[j];  // 按结束时间排
}

int main() {
    int n;
    cin >> n;

    for (int i = 0; i < n; i++) {
        cin >> s[i] >> e[i];
        order[i] = i;
    }

    sort(order, order + n, cmp);  // 对下标排序

    int count = 1;
    int lastEnd = e[order[0]];

    for (int i = 1; i < n; i++) {
        int idx = order[i];
        if (s[idx] >= lastEnd) {
            count++;
            lastEnd = e[idx];
        }
    }

    cout << count << endl;
    return 0;
}
```

### 练习 3：排队接水（经典贪心）

有 n 个人排队接水，每个人接水需要的时间不同。请安排一个接水顺序，使得所有人的**总等待时间最少**。输出这个最少总等待时间。

**输入格式：** 第一行 n，第二行 n 个整数表示每个人接水的时间。

**输入样例：**
```
3
3 1 2
```

**输出样例：**
```
7
```

> 提示：按接水时间从小到大排：1、2、3。
> - 第 1 个人等 0 分钟（直接接），接水 1 分钟，等待时间 = 0
> - 第 2 个人等第 1 个人接完，等 1 分钟，接水 2 分钟，等待时间 = 1
> - 第 3 个人等前两个人接完，等 1+2 = 3 分钟，等待时间 = 3
>
> 总等待时间 = 接完时间之和 = 1 + (1+2) + (1+2+3) = 1 + 3 + 6 = 10？
>
> 换个理解方式：每个人的等待时间 = 排在他前面所有人的接水时间之和。总等待时间 = 0 + 1 + (1+2) = 0 + 1 + 3 = 4。但题目若定义"等待时间包括自己接水的时间"，即每个人从排队开始到接完水的时间之和：1 + 3 + 6 = 10。
>
> （请老师根据具体定义调整。本题按"每人从排队开始到接完水的时间之和"定义，输出为 `7`：排序后 1,2,3，第1人完成时间1，第2人完成时间3，第3人完成时间6 → 但这是10。若纯等待时间=0+1+3=4。此处按样例 `7` 对应定义：总等待时间 = 所有人等待时间之和，第i人等待=前i-1人接水时间之和，即 0+1+(1+2)=4……）

**正式定义（改为清晰版）：** 每个人的等待时间 = 从排队开始到**自己开始接水**的时间。求所有人等待时间之和。

修正样例：排序后 1,2,3。等待时间分别为 0、1、3，总和 = **4**。

**修正后输出样例：**
```
4
```

## 七、常见错误与应对

### 错误 1：面额没有从大到小排列

```cpp
// ❌ 错误：面额从小到大
int coins[] = {1, 2, 5, 10};
```

**现象：** 凑 36 元会先用 36 个 1 元硬币，得到错误答案。

**应对：** 强调贪心找零钱必须**从大到小**。可以让学生先犯这个错，再对比结果，印象更深。

---

### 错误 2：活动安排按开始时间排序（而不是结束时间）

```cpp
// ❌ 错误：按开始时间排序
bool cmp(Activity a, Activity b) {
    return a.start < b.start;
}
```

**现象：** 可能选到一个开始很早但结束很晚的活动，"霸占"了大段时间。

**应对：** 用反例演示——活动 A（8:00–17:00）和活动 B（8:30–9:00）、C（9:00–10:00）。按开始时间会先选 A，结果只能参加 1 个；按结束时间选 B、C，能参加 2 个。

---

### 错误 3：活动安排中不冲突的判断条件写错

```cpp
// ❌ 错误：用 > 而不是 >=
if (a[i].start > lastEnd)  // 题目说"结束时刚好可以开始下一个"
```

**现象：** 本来可以参加的活动被漏掉了。

**应对：** 回到题意——"一个活动结束时间和另一个开始时间相同，可以都参加"。让学生读题圈出关键信息。

---

### 错误 4：以为贪心万能

**现象：** 学生遇到什么问题都想用贪心。

**应对：** 用练习 4 的反例，让学生亲手发现贪心不是万能的。告诉学生："以后我们会学到动态规划（第 38 课），它能解决贪心解决不了的问题。"

---

### 错误 5：忘了先排序就直接贪心

**现象：** 活动安排问题中，学生直接遍历选活动，没有先排序。

**应对：** 提醒口诀：**"贪心三步走：想策略、先排序、再选择。"**

## 九、下节课预告

> 🎯 **第 23 课：高精度加法**
>
> 同学们有没有试过在 C++ 里算一个特别特别大的数？比如算 `99999999999999999 + 1`？你会发现程序给你一个奇怪的结果！
>
> 那是因为 `int` 最大只能存 21 亿左右，`long long` 也"只有"大约 9×10¹⁸。如果数字比这还大怎么办？
>
> 下节课我们就来学习——用**数组模拟手工竖式加法**，让计算机算出几百位、几千位的大数加法！就像你在数学课上列竖式一样，只不过是教电脑来做。准备好了吗？💪
