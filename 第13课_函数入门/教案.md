## 二、教学重点与难点

### 重点

- 函数的完整语法结构：`返回类型 函数名(参数列表) { 函数体; return 值; }`
- 函数的调用过程：传参 → 执行 → 返回
- `void` 函数与有返回值函数（`int`、`double`）的区别

### 难点

- **参数的"副本"机制**（值传递）：参数传进函数后，函数里改了不影响外面。小学生容易以为"传进去就变了"。
- **`return` 的含义**：既是"交出结果"，又是"函数到此结束"。
- **函数定义的位置**：必须写在 `main` 前面（本课阶段只讲这一种方式，暂不讲函数声明）。

## 四、讲稿提纲

### 4.1 开场故事（5 分钟）

> 同学们，想象一下——你开了一家"果汁店"🍹。每来一个顾客，你都要：洗水果 → 切水果 → 放进榨汁机 → 倒进杯子 → 递给顾客。
>
> 如果来了 100 个顾客，你总不能把这五步写 100 遍吧？
>
> 所以我们买了一台**榨汁机**！只要把水果放进去（这就是**参数**），按一下按钮（这就是**调用**），它就会自动帮你做好果汁吐出来（这就是**返回值**）。
>
> 今天，我们要学的"函数"，就是程序里的**榨汁机**——一个你可以反复使用的**小帮手**！

**互动提问**：
- "如果我放进去苹果，出来的是什么？"——苹果汁（不同的参数，不同的结果）
- "如果我不需要它吐出果汁，只需要它帮我把水果切好呢？"——引出 `void`（只做事，不给东西回来）

### 4.2 函数的语法结构（15 分钟）

**板书/PPT 展示函数模板：**

```
返回类型 函数名(参数) {
    做的事情;
    return 结果;
}
```

**用榨汁机对照：**

| 函数组成 | 榨汁机对照 | 举例 |
|----------|-----------|------|
| 返回类型 | 出来的是什么类型的产品 | `int`（整数果汁）、`double`（小数果汁）、`void`（不出东西） |
| 函数名 | 这台机器叫什么名字 | `zhajizhi`、`add`、`maxVal` |
| 参数 | 你放进去的材料 | `int a, int b`（两个整数水果） |
| return | 把产品递出来 | `return a + b;` |

**关键口诀（带学生念一遍）：**

> 先写"出什么"，再写"叫什么"，括号里写"吃什么"，大括号里写"做什么"，最后 `return` "吐什么"。

**第一个函数演示（边讲边写）：**

```cpp
#include <iostream>
using namespace std;

// 这是我们的"加法小帮手"
int add(int a, int b) {
    int sum = a + b;
    return sum;
}

int main() {
    int result = add(3, 5);
    cout << result << endl;  // 输出 8
    return 0;
}
```

**讲解要点：**
- `add` 写在 `main` 上面——"帮手要先招聘好，才能在工作中叫他"
- `add(3, 5)` 就是"叫小帮手来干活"，3 和 5 是给他的材料
- `return sum` 就是小帮手把结果递给你
- `int result = add(3, 5)` —— 你用一个变量接住他递回来的结果

### 4.3 void 函数 + 值传递（15 分钟）

**void 函数讲解：**

> 有些小帮手，你叫他做事，他做完就行了，不需要递东西给你。比如你让一个小帮手"去黑板上写字"，他写完就好了，不用递什么东西回来。这种小帮手的类型就是 `void`——"什么都不返回"。

**值传递——"抄作业"比喻：**

> 当你调用函数的时候，参数就像是"抄了一份作业给小帮手"。小帮手拿到的是一份**副本**，他在副本上怎么涂改，都不会影响你原来的那份作业。

```cpp
#include <iostream>
using namespace std;

void tryToChange(int x) {
    x = 100;  // 小帮手在自己的副本上写了100
    cout << "函数里面: x = " << x << endl;
}

int main() {
    int a = 1;
    tryToChange(a);
    cout << "函数外面: a = " << a << endl;  // a 还是 1！
    return 0;
}
```

> 看到了吗？小帮手把他的副本改成了 100，但你原来的 `a` 还是 1。这就叫**值传递**——传的是值的副本，不是原件！

### 4.4 课堂小结口诀（5 分钟）

带学生一起念：

> **函数四要素，记牢不会错：**
> **返回写前面，名字自己取，**
> **括号装参数，大括号写过程，**
> **return 交结果，void 不用还。**
> **调用写名字，括号填材料，**
> **帮手定义在前，main 里面才能用！**

## 五、课堂例题

### 例题 1：最简单的函数——求两数之和（⭐）

**题目**：写一个函数 `add`，接收两个整数，返回它们的和。在 `main` 中输入两个数，调用函数并输出结果。

**讲解思路**：这是第一个函数，带着学生从零开始写。强调"先定义、后调用"的顺序。

```cpp
#include <iostream>
using namespace std;

int add(int a, int b) {
    return a + b;
}

int main() {
    int x, y;
    cin >> x >> y;
    cout << add(x, y) << endl;
    return 0;
}
```

**讲解要点**：
- `int add(int a, int b)` —— 出来是整数，名叫 add，吃两个整数
- `return a + b;` —— 可以直接 return 一个表达式，不一定要先存到变量里
- `add(x, y)` —— 调用时，x 的值会"抄"给 a，y 的值会"抄"给 b

---

### 例题 2：求三个数中的最大值（⭐⭐）

**题目**：写一个函数 `maxTwo`，接收两个整数，返回较大的那个。然后在 `main` 中利用它求出三个数中的最大值。

**讲解思路**：展示函数可以被多次调用，以及函数的返回值可以作为另一个函数调用的参数。

```cpp
#include <iostream>
using namespace std;

int maxTwo(int a, int b) {
    if (a > b) {
        return a;
    } else {
        return b;
    }
}

int main() {
    int a, b, c;
    cin >> a >> b >> c;

    // 先比前两个，再拿赢家和第三个比
    int ans = maxTwo(maxTwo(a, b), c);
    cout << ans << endl;
    return 0;
}
```

**讲解要点**：
- `return` 出现了两次——走到哪个 `return` 就从那里结束函数
- `maxTwo(maxTwo(a, b), c)` —— "套娃"调用：先让内层帮手比 a 和 b，拿到结果再让外层帮手和 c 比
- 问学生："如果输入 3 7 5，程序执行顺序是什么？"——先算 `maxTwo(3, 7)` 得到 7，再算 `maxTwo(7, 5)` 得到 7

---

### 例题 3：用 void 函数打印一排星号（⭐）

**题目**：写一个 `void` 函数 `printStars`，接收一个整数 `n`，打印一行 `n` 个星号。然后在 `main` 中调用它打印一个直角三角形。

**讲解思路**：让学生理解 void 函数"只做事、不返回"。同时复习循环。

```cpp
#include <iostream>
using namespace std;

void printStars(int n) {
    for (int i = 0; i < n; i++) {
        cout << "*";
    }
    cout << endl;  // 打完一行要换行
    // 注意：void函数不需要写return（写了也行，但不带值）
}

int main() {
    int rows;
    cin >> rows;
    for (int i = 1; i <= rows; i++) {
        printStars(i);  // 第1行1个星，第2行2个星……
    }
    return 0;
}
```

**输入示例**：`5`

**输出示例**：
```
*
**
***
****
*****
```

**讲解要点**：
- `void` —— 这个函数不 return 东西，它的任务就是"打印"
- 调用的时候不能写 `int x = printStars(3);`，因为它没有返回值，没东西可接
- 函数帮我们把"打印一排星号"这件事封装了，`main` 里只要说"打几个"就行

---

### 例题 4：判断一个数是不是质数（⭐⭐⭐）

**题目**：写一个函数 `isPrime`，接收一个整数，如果是质数返回 `true`（可以用 `1` 代替），否则返回 `false`（用 `0` 代替）。然后在 `main` 中输入一个整数 n，输出 2 到 n 之间所有的质数。

**讲解思路**：综合例题，展示函数在实际问题中的威力——把复杂的判断逻辑封装起来，`main` 里面清清爽爽。

```cpp
#include <iostream>
using namespace std;

// 判断 x 是不是质数
// 质数：大于1，且只能被1和自己整除
int isPrime(int x) {
    if (x < 2) {
        return 0;  // 不是质数
    }
    for (int i = 2; i * i <= x; i++) {
        if (x % i == 0) {
            return 0;  // 能被整除，不是质数
        }
    }
    return 1;  // 没人能整除它，是质数
}

int main() {
    int n;
    cin >> n;
    for (int i = 2; i <= n; i++) {
        if (isPrime(i) == 1) {
            cout << i << " ";
        }
    }
    cout << endl;
    return 0;
}
```

**输入示例**：`20`

**输出示例**：`2 3 5 7 11 13 17 19`

**讲解要点**：
- 用 `int` 返回 0/1 来代替 true/false（四年级暂不深入 `bool` 类型，知道即可）
- `i * i <= x` 是优化技巧——不需要查到 x，查到 √x 就够了（可以简单解释：如果一个数能拆成两个因数相乘，其中至少一个不超过它的平方根）
- **重点感受**：如果没有函数，`main` 里面要写两层嵌套循环 + 判断逻辑，读起来很乱。有了 `isPrime` 函数，`main` 里面一目了然——"是质数就输出"

### 练习 4：消灭重复代码（⭐⭐⭐）

**题目**：下面这段代码可以正常运行，但有很多重复。请你把重复部分提取成一个函数，让代码变简洁。

```cpp
#include <iostream>

int main() {
    // 求 1 到 5 的和
    int sum1 = 0;
    for (int i = 1; i <= 5; i++) {
        sum1 += i;
    }
    cout << sum1 << endl;

    // 求 1 到 10 的和
    int sum2 = 0;
    for (int i = 1; i <= 10; i++) {
        sum2 += i;
    }
    cout << sum2 << endl;

    // 求 1 到 100 的和
    int sum3 = 0;
    for (int i = 1; i <= 100; i++) {
        sum3 += i;
    }
    cout << sum3 << endl;

    return 0;
}
```

**要求**：定义一个函数 `sumTo(int n)`，返回 1 到 n 的和，然后在 `main` 中调用三次。

## 七、常见错误与应对

### 错误 1：函数定义写在 main 后面

```cpp
int main() {
    cout << add(3, 5);  // ❌ 编译器说"add 是什么？我不认识！"
    return 0;
}

int add(int a, int b) {
    return a + b;
}
```

**应对**：告诉学生——"帮手要先招聘，才能干活。写在 `main` 上面！"本课阶段统一要求函数定义写在 `main` 之前，后续再学函数声明（前置声明）。

---

### 错误 2：忘记写 return

```cpp
int add(int a, int b) {
    int sum = a + b;
    // 忘记写 return sum; 了！
}
```

**应对**：编译器可能警告也可能不警告，但结果是未定义的（输出乱七八糟的数）。提醒学生——"帮手做完事忘了把结果交给你，你收到的就是一堆乱码！**有返回类型的函数，一定要写 return！**"

---

### 错误 3：调用时重新写了类型

```cpp
int main() {
    int result = add(int a, int b);  // ❌ 调用的时候不要写类型！
}
```

**应对**：区分"定义"和"调用"——"招聘的时候要写简历（类型 + 名字），叫他干活的时候直接喊名字就行了！"
- 定义：`int add(int a, int b)` ✅
- 调用：`add(3, 5)` ✅ 、`add(x, y)` ✅

---

### 错误 4：void 函数里写了 return 值

```cpp
void sayHello() {
    cout << "Hello!" << endl;
    return 1;  // ❌ void 函数不能 return 一个值！
}
```

**应对**："`void` 的意思是'空的'——这个帮手不给你递东西。你让他 `return 1`，他一脸懵——'你说好不用我递东西的呀！'"

---

### 错误 5：以为函数里改了参数，外面也会变

```cpp
void changeIt(int x) {
    x = 999;
}

int main() {
    int a = 1;
    changeIt(a);
    cout << a << endl;  // 学生期望输出 999，实际输出 1
}
```

**应对**：重复"抄作业"比喻——"你把作业抄了一份给帮手，帮手在副本上写了 999，你手上的原件当然不变啊！"
