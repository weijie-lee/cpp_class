## 二、教学重点与难点

### 重点

- 选择排序的核心思路：每轮从"未排序区"找最小值，放到"已排序区"末尾
- 插入排序的核心思路：依次取"未排序区"第一张牌，插入"已排序区"正确位置
- 三种排序算法（冒泡、选择、插入）的思路对比

### 难点

- 选择排序中"记录最小值下标"而不是记录最小值本身（先记位置，最后再交换）
- 插入排序中"边比较边挪动"的过程（`while` 循环从后往前挪元素）
- 插入排序中 `temp` 变量的作用：为什么要先把牌抽出来保存

## 四、讲稿提纲

### 4.1 热身回顾（5 min）

> 同学们，上节课我们学了第一种排序方法——**冒泡排序**。还记得冒泡排序是怎么干活的吗？
>
> 对！冒泡排序就像水里的气泡一样，**大的泡泡一点一点往上冒**。每一轮，相邻的两个数比一比，大的往后挪，一轮下来最大的数就"冒"到了最后面。
>
> 快速抢答：
> - 冒泡排序需要几层循环？（两层 `for`）
> - 每一轮结束后，哪个位置的数已经确定了？（最后面那个，是当前最大的）
> - 5 个数最多需要几轮？（4 轮）
>
> 很好！今天我们要学**两种全新的排序方法**——**选择排序**和**插入排序**。它们和冒泡排序**目的相同**（把数从小到大排好），但**干活的思路完全不同**！就像去学校有三条路，终点一样，走法不同。

### 4.2 讲解①：选择排序（20 min）

#### 故事引入——体育课排队

> 同学们上体育课排队的时候，体育老师是怎么排的？
>
> 想象一下：体育老师让全班 30 个同学站成一排，然后开始排队。老师的方法是——
>
> **第 1 步**：老师用眼睛"扫"一遍所有人，找到**最矮的那个同学**，让他走出来，站到**第 1 个位置**。
> **第 2 步**：老师再"扫"一遍剩下的人，找到**剩下人里最矮的**，站到**第 2 个位置**。
> **第 3 步**：再扫一遍剩下的……
> ……
> **一直重复**，直到所有人都站好了！
>
> 这就是**选择排序**的思路：**每一轮，从还没排好的部分里"选"出最小的，放到正确位置**。

#### 关键概念

> 我们把数组想象成两个区域：
> - 左边是**"已排好区"**（站好的同学）
> - 右边是**"未排好区"**（还在乱站的同学）
>
> 一开始，所有人都在"未排好区"。每做一轮，就从未排好区挑一个最小的，放到已排好区的最后面。已排好区就多了一个人，未排好区就少了一个人。

#### 板书/PPT 演示

用具体数字演示过程：`a[] = {5, 3, 8, 1, 4}`

| 轮次 | 未排好区 | 找到最小值 | 交换操作 | 排完后数组 |
|------|---------|-----------|---------|-----------|
| 第 1 轮 | `[5, 3, 8, 1, 4]` | 1（下标 3） | `a[0]` ↔ `a[3]` | **1**, 3, 8, 5, 4 |
| 第 2 轮 | `[3, 8, 5, 4]` | 3（下标 1） | 已在正确位置 | 1, **3**, 8, 5, 4 |
| 第 3 轮 | `[8, 5, 4]` | 4（下标 4） | `a[2]` ↔ `a[4]` | 1, 3, **4**, 5, 8 |
| 第 4 轮 | `[5, 8]` | 5（下标 3） | 已在正确位置 | 1, 3, 4, **5**, 8 |

> 4 轮结束，5 个数排好了！注意：**5 个数只需要 4 轮**（最后一个不用选，自然就是最大的）。

#### 代码讲解要点

> 选择排序有一个很聪明的小技巧：我们不是找到最小的数就搬过来，而是**先记住最小数的"位置"（下标）**，最后再交换。
>
> 为什么呢？就像体育老师不是看到一个矮的就让他往前挤（那会很乱），而是**先用手指着最矮的那个人**，全扫完了再让他走到前面。
>
> 我们用一个变量 `minIdx` 来"指着"当前最小的人的位置。

#### 易错点强调

> ⚠️ 注意：是记录**下标**（位置），不是记录值！
> - `minIdx = i;` ✅ 记住了"第几号位置"
> - `minVal = a[i];` ❌ 只记了值，不知道在哪，没法交换！
>
> 口诀：**"选择排序记下标，找到位置再交换"**。

### 4.4 讲解②：插入排序（20 min）

#### 故事引入——整理扑克牌

> 同学们打过扑克牌吗？发牌的时候，你们是怎么整理手牌的？
>
> 是不是这样：一开始手上没有牌。每抓到一张新牌，你就在手里**从右往左**找一个合适的位置，把新牌**插进去**。比如：
>
> - 手上是空的，抓到 **7**，直接拿着：`[7]`
> - 抓到 **3**，3 比 7 小，插到 7 前面：`[3, 7]`
> - 抓到 **5**，5 比 7 小、比 3 大，插到中间：`[3, 5, 7]`
> - 抓到 **2**，2 比所有都小，插到最前面：`[2, 3, 5, 7]`
>
> 你看，每抓一张牌，手里的牌**始终是排好序的**！这就是**插入排序**的思路。

#### 对应到数组

> 在数组里，我们假设 `a[0]` 一开始就是"手里的第一张牌"（一张牌当然是有序的）。
>
> 然后从 `a[1]` 开始，每个元素都是"新抓到的牌"：
> 1. 先把新牌**抽出来**，存到一个临时变量 `temp` 里（不然会被覆盖！）
> 2. 从这张牌的**前一个位置开始往左看**
> 3. 如果左边的牌**比新牌大**，就把它**往右挪一格**（给新牌腾位置）
> 4. 直到找到一个**不比新牌大**的位置，把新牌**插进去**！

#### 板书/PPT 演示

用具体数字演示过程：`a[] = {5, 3, 8, 1, 4}`

**第 1 轮**：新牌 = `a[1]` = 3，`temp = 3`
- 看 `a[0]` = 5，5 > 3，把 5 往右挪 → 数组变成 `[5, 5, 8, 1, 4]`
- 左边没牌了，把 3 插到位置 0 → 数组变成 `[3, 5, 8, 1, 4]`

**第 2 轮**：新牌 = `a[2]` = 8，`temp = 8`
- 看 `a[1]` = 5，5 < 8，不用挪！8 就待在原位 → `[3, 5, 8, 1, 4]`

**第 3 轮**：新牌 = `a[3]` = 1，`temp = 1`
- 看 `a[2]` = 8，8 > 1，挪 → `[3, 5, 8, 8, 4]`
- 看 `a[1]` = 5，5 > 1，挪 → `[3, 5, 5, 8, 4]`
- 看 `a[0]` = 3，3 > 1，挪 → `[3, 3, 5, 8, 4]`
- 左边没牌了，插到位置 0 → `[1, 3, 5, 8, 4]`

**第 4 轮**：新牌 = `a[4]` = 4，`temp = 4`
- 看 `a[3]` = 8，8 > 4，挪 → `[1, 3, 5, 8, 8]`
- 看 `a[2]` = 5，5 > 4，挪 → `[1, 3, 5, 5, 8]`
- 看 `a[1]` = 3，3 < 4，停！插到位置 2 → `[1, 3, 4, 5, 8]`

> 排好了！✨

#### 关键理解

> 插入排序有两个要点：
> 1. **`temp` 变量是"安全口袋"**：把新牌先放进口袋里保护好，不然在挪动的时候它会被覆盖掉！
> 2. **挪动方向是从右往左**：用 `while` 循环，`j` 从当前位置前一个开始往左走，碰到比 `temp` 大的就往右推一格。

### 4.5 讲解③：三种排序大比拼（12 min）

> 现在我们已经学了三种排序方法了！来做一个对比：

| 对比项 | 冒泡排序 | 选择排序 | 插入排序 |
|--------|---------|---------|---------|
| 比喻 | 大泡泡往上冒 | 体育老师挑最矮的 | 整理扑克牌 |
| 核心操作 | 相邻两两比较交换 | 找最小值放到前面 | 把新元素插到合适位置 |
| 交换次数 | 多（每次比较都可能交换） | 少（每轮最多交换一次） | 不是交换，是挪动+插入 |
| 特点 | 最好理解 | 交换次数最少 | 对基本有序的数据很快 |
| 时间复杂度 | O(n²) | O(n²) | O(n²) |

> 三种排序，**目的相同，思路不同**！就像去学校可以走大路、小路、或者骑自行车，终点一样，走法不同。
>
> 有同学可能会问："那到底用哪种好？"答案是——**都可以！** 而且下节课我们还会学一个超级厉害的武器 `sort` 函数，一行代码就搞定排序，比这三种都快！但是，**理解这三种排序的思路非常重要**，因为竞赛题里经常考排序的思想。

## 五、课堂例题

### 例题 1：选择排序——从小到大排队

**题目**：输入 n 个整数，使用**选择排序**将它们从小到大排列并输出。

**输入样例**：
```
5
5 3 8 1 4
```
**输出样例**：
```
1 3 4 5 8
```

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    int a[1005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    // 选择排序：每一轮从未排好的部分选出最小值
    for (int i = 0; i < n - 1; i++) {
        int minIdx = i;  // 先假设"未排好区"第一个就是最小的
        // 在未排好区里找真正的最小值
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[minIdx]) {
                minIdx = j;  // 发现更小的，更新下标
            }
        }
        // 把最小值交换到位置 i（已排好区的末尾）
        if (minIdx != i) {  // 如果最小值不在位置 i，才需要交换
            int temp = a[i];
            a[i] = a[minIdx];
            a[minIdx] = temp;
        }
    }

    // 输出排好的数组
    for (int i = 0; i < n; i++) {
        if (i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;

    return 0;
}
```

**讲解思路**：
1. 外层循环 `i` 从 0 到 `n-2`：表示当前要确定第 `i` 个位置的值（共需要 `n-1` 轮）。
2. 每轮开始，先假设 `minIdx = i`，即"未排好区第一个"是最小的。
3. 内层循环 `j` 从 `i+1` 到 `n-1`：在未排好区里找有没有更小的。如果 `a[j] < a[minIdx]`，就更新 `minIdx = j`。
4. 内层循环结束后，`minIdx` 就是真正最小值的位置。用三步交换法把 `a[i]` 和 `a[minIdx]` 交换。
5. 外层循环结束，数组就排好了！

> 💡 **类比记忆**：外层循环 = 老师叫"第几个位置站好"，内层循环 = 老师在剩下的人里扫一遍找最矮的。

### 例题 2：插入排序——扑克牌整理

**题目**：输入 n 个整数，使用**插入排序**将它们从小到大排列并输出。

**输入样例**：
```
5
5 3 8 1 4
```
**输出样例**：
```
1 3 4 5 8
```

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;

    int a[1005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    // 插入排序：从第 2 张牌开始，每张牌插到前面合适的位置
    for (int i = 1; i < n; i++) {
        int temp = a[i];  // 把新牌抽出来放进"安全口袋"
        int j = i - 1;     // 从新牌前一个位置开始往左看
        // 往左找位置：如果左边的牌比新牌大，就往右挪一格
        while (j >= 0 && a[j] > temp) {
            a[j + 1] = a[j];  // 把大牌往右推
            j--;                // 继续往左看
        }
        a[j + 1] = temp;  // 找到合适位置，把新牌插进去！
    }

    // 输出排好的数组
    for (int i = 0; i < n; i++) {
        if (i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;

    return 0;
}
```

**讲解思路**：
1. 外层循环 `i` 从 1 开始（第 0 张牌不用插，它自己就是"有序的一张牌"）。
2. `temp = a[i]`：把新牌抽出来保护好。**这一步千万不能漏！**
3. `j = i - 1`：从新牌的左边第一个位置开始。
4. `while` 循环做两件事：
   - 检查 `j >= 0`（还没走到数组最左边）
   - 检查 `a[j] > temp`（左边的牌比新牌大）
   - 两个条件都满足，就把 `a[j]` 往右挪一格（`a[j+1] = a[j]`），然后 `j--` 继续往左看。
5. `while` 结束时，`j+1` 就是新牌应该插入的位置。
6. `a[j+1] = temp`：把新牌插进去！

> 💡 **为什么是 `j+1` 而不是 `j`？** 因为 `while` 循环结束时，`j` 已经多退了一步（要么 `j < 0` 了，要么 `a[j]` 不比 `temp` 大了），所以新牌应该放在 `j` 的右边，即 `j+1`。

### 例题 3：三种排序都会写——排序小能手

**题目**：输入 n 个整数和一个数字 k（k 为 1、2 或 3）。当 k=1 时用冒泡排序，k=2 时用选择排序，k=3 时用插入排序，将数组从小到大排列并输出。

**输入样例**：
```
5 2
4 2 7 1 3
```
**输出样例**：
```
1 2 3 4 7
```

**完整代码**：

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, k;
    cin >> n >> k;

    int a[1005];
    for (int i = 0; i < n; i++) {
        cin >> a[i];
    }

    if (k == 1) {
        // 冒泡排序（上节课学的）
        for (int i = 0; i < n - 1; i++) {
            for (int j = 0; j < n - 1 - i; j++) {
                if (a[j] > a[j + 1]) {
                    int temp = a[j];
                    a[j] = a[j + 1];
                    a[j + 1] = temp;
                }
            }
        }
    } else if (k == 2) {
        // 选择排序
        for (int i = 0; i < n - 1; i++) {
            int minIdx = i;
            for (int j = i + 1; j < n; j++) {
                if (a[j] < a[minIdx]) {
                    minIdx = j;
                }
            }
            if (minIdx != i) {
                int temp = a[i];
                a[i] = a[minIdx];
                a[minIdx] = temp;
            }
        }
    } else {
        // 插入排序
        for (int i = 1; i < n; i++) {
            int temp = a[i];
            int j = i - 1;
            while (j >= 0 && a[j] > temp) {
                a[j + 1] = a[j];
                j--;
            }
            a[j + 1] = temp;
        }
    }

    // 输出
    for (int i = 0; i < n; i++) {
        if (i > 0) cout << " ";
        cout << a[i];
    }
    cout << endl;

    return 0;
}
```

**讲解思路**：
1. 这道题是一道**三合一综合题**！三种排序方法用 `if-else` 选择。
2. 重点不是这道题本身多难，而是让同学们**把三种排序的代码都写一遍**，加深肌肉记忆。
3. 引导学生观察：三段代码虽然长得不同，但**排序结果完全一样**——条条大路通罗马！

> 🏆 **成就感时刻**：同学们，你们现在已经掌握了**三种排序方法**了！在很多初中生甚至高中生还搞不清楚排序的时候，你们已经能写三种了，是不是很骄傲？

### 练习 3：排序并去重

## 七、常见错误与应对

### 错误 1：选择排序中记录值而不是下标

```cpp
int minVal = a[i];  // ❌ 只记了值
for (int j = i + 1; j < n; j++) {
    if (a[j] < minVal) {
        minVal = a[j];  // 记了新的最小值，但不知道它在哪！
    }
}
// 接下来怎么交换？不知道 minVal 对应的下标是几！
```

**正确做法**：
```cpp
int minIdx = i;  // ✅ 记住下标（位置）
for (int j = i + 1; j < n; j++) {
    if (a[j] < a[minIdx]) {
        minIdx = j;  // 更新最小值的下标
    }
}
// 现在用 a[i] 和 a[minIdx] 交换就行了
```

**口诀**：**"选择排序记位置，知道在哪才能换"**。

---

### 错误 2：插入排序忘记保存 temp

```cpp
for (int i = 1; i < n; i++) {
    int j = i - 1;
    while (j >= 0 && a[j] > a[i]) {  // ❌ a[i] 已经被覆盖了！
        a[j + 1] = a[j];
        j--;
    }
    a[j + 1] = a[i];  // ❌ a[i] 的值早就变了！
}
```

**问题**：当 `a[j]` 往右挪到 `a[j+1]` 时，如果 `j+1` 刚好是 `i`，那 `a[i]` 原来的值就被覆盖了！

**正确做法**：
```cpp
int temp = a[i];  // ✅ 先把新牌放进安全口袋
int j = i - 1;
while (j >= 0 && a[j] > temp) {  // ✅ 用 temp 比较
    a[j + 1] = a[j];
    j--;
}
a[j + 1] = temp;  // ✅ 用 temp 插入
```

**口诀**：**"新牌先进口袋里，不然挪着挪着就丢了"**。

---

### 错误 3：插入排序 while 条件顺序写反

```cpp
while (a[j] > temp && j >= 0) {  // ❌ 危险！j 可能已经是 -1 了
```

**问题**：当 `j = -1` 时，先访问 `a[-1]`，这是数组越界！C++ 不会报错但结果不可预测。

**正确做法**：
```cpp
while (j >= 0 && a[j] > temp) {  // ✅ 先检查 j 合法，再访问数组
```

**口诀**：**"先看下标合不合法，再看数值大不大"**（短路求值：`j >= 0` 不满足时，后面的 `a[j]` 根本不会执行）。

---

### 错误 4：选择排序外层循环写成 i < n

```cpp
for (int i = 0; i < n; i++) {  // ❌ 多循环了一轮
```

**问题**：最后一轮只剩一个数，不需要再"选"了。多循环一轮虽然结果不会错（自己跟自己比），但浪费时间。

**正确做法**：
```cpp
for (int i = 0; i < n - 1; i++) {  // ✅ n 个数只需要 n-1 轮
```

**口诀**：**"n 个数排 n-1 轮，最后一个自然归位"**。

---

### 错误 5：交换两个数时漏了 temp

```cpp
a[i] = a[minIdx];
a[minIdx] = a[i];  // ❌ 此时 a[i] 已经变了，两个变成一样的了！
```

**正确做法**：
```cpp
int temp = a[i];      // ✅ 三步交换法
a[i] = a[minIdx];
a[minIdx] = temp;
```

**口诀**：**"交换要靠小助手，三步交换不能少"**（这个上节课冒泡排序也强调过！）。
