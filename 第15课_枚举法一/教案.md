### 教学目标（可检验）

1. **能说出枚举法的核心思想**：用循环把所有可能的答案一个一个试出来，再用 `if` 判断哪些满足条件。
2. **能独立写出"找满足条件的数"类枚举程序**：给定范围和条件，正确使用 `for` + `if` 枚举并输出答案（如水仙花数、完全平方数）。
3. **初步具备"缩小枚举范围"的优化意识**：能判断哪些值不可能是答案，从而减少循环次数。

## 二、教学重点与难点

### 重点

- 枚举法的基本模式：**确定范围 → 逐个尝试 → 判断条件 → 收集答案**。
- 用 `for` 循环 + `if` 判断实现枚举。
- 百钱买百鸡问题（简化版）的建模与求解。

### 难点

- 从题目文字中提取"枚举什么"和"满足什么条件"。
- 多重循环枚举时，理解每层循环的含义。
- 优化意识：缩小枚举范围（上界的确定）。

## 四、讲稿提纲

### 4.1 热身导入（0′–5′）——"找钥匙"的故事

> 同学们，假设你有一串钥匙，上面挂了 10 把钥匙🔑，但你忘了哪把能打开家门。你会怎么办？
>
> 对！**一把一把试**！从第 1 把开始，插进锁孔，不行就换第 2 把、第 3 把……直到找到能开门的那一把。
>
> 这种方法，在编程里有一个很酷的名字，叫做——**枚举法**！
>
> "枚"是"一个一个"的意思，"举"是"列出来"的意思。
>
> 枚举法就是：**把所有可能的答案，一个一个列出来试，看看哪个是对的。**

### 4.2 新课讲解 ①（5′–20′）——枚举法的基本模式

#### 再来一个比喻：猜密码

> 想象你同学的自行车锁是三位数密码，从 000 到 999。你忘了密码，怎么办？
>
> 最笨但最靠谱的方法：从 000 开始一个一个试，000、001、002……一直到 999，总有一个能打开！
>
> 这就是枚举法的思想——**不怕慢，只要一个不落地全部试过，答案一定能找到。**

#### 枚举法三步走（板书）

```
第一步：确定范围 —— 答案可能在哪里？（从几到几？）
第二步：逐个尝试 —— 用 for 循环一个一个试
第三步：判断条件 —— 用 if 检查当前这个是不是答案
```

#### 代码基本模板

```cpp
for (int i = 起点; i <= 终点; i++) {
    if (满足条件) {
        // 找到了！输出或记录
    }
}
```

> 看！其实枚举法就是我们学过的 `for` 循环和 `if` 判断的组合拳，一点也不难。

### 4.3 例题 1 讲解（20′–28′）——找完全平方数

> 什么是完全平方数？就是某个整数乘以自己得到的数。比如 1×1=1，2×2=4，3×3=9……
>
> 现在问题来了：1 到 100 之间，有哪些完全平方数？
>
> 我们可以用枚举法：让 i 从 1 跑到 100，检查 i 是不是完全平方数。
>
> 怎么检查呢？如果 i 开根号是整数，那它就是完全平方数。

（代码见第五板块例题 1）

### 4.4 例题 2 讲解（35′–45′）——水仙花数

> 接下来我们挑战一个更好玩的：**水仙花数**。
>
> 什么是水仙花数？就是一个三位数，它的每一位数字的立方（自己乘自己再乘自己）加起来，等于它本身。
>
> 比如 153：1³ + 5³ + 3³ = 1 + 125 + 27 = 153。哇，真的等于它自己！
>
> 那三位数有多少个？从 100 到 999，一共 900 个。我们让电脑一个一个去试！

### 4.5 例题 3 讲解（60′–70′）——百钱买百鸡

> 古代有一道经典数学题：
>
> 公鸡 5 元一只，母鸡 3 元一只，小鸡 1 元买 3 只。现在有 100 元钱，要买 100 只鸡，每种鸡至少买 1 只。问有多少种买法？
>
> 这道题就像在超市挑东西——我们要把公鸡的数量、母鸡的数量都一个一个去试。
>
> 但是注意：**公鸡最多买多少只？** 100 元，5 元一只，最多 20 只。但还要留钱买母鸡和小鸡，而且至少各 1 只，所以公鸡最多买 (100-3-1)÷5 = 19 只，不过我们取上界 20 也行，反正不满足条件就跳过。
>
> 母鸡呢？最多 33 只。
>
> 关键来了：**知道了公鸡和母鸡的数量，小鸡的数量还用枚举吗？不用！** 因为总共 100 只鸡，小鸡 = 100 - 公鸡 - 母鸡，直接算就行了！
>
> 这就是我们今天要学的一个小技巧——**能算的就不枚举，能少枚举一层就少一层。**

（代码见第五板块例题 3）

## 五、课堂例题

### 例题 1：找出 1 到 N 之间的所有完全平方数

**题目**：输入一个正整数 N（N ≤ 10000），输出 1 到 N 之间所有的完全平方数，每行一个。

**讲解思路**：
1. 枚举什么？——枚举 1 到 N 的每一个数 i。
2. 条件是什么？——i 是完全平方数，即 i 开根号后取整再平方等于 i。
3. 用 `(int)sqrt(i)` 取整后再平方，和 i 比较。

**方法一（枚举每个数去判断）**：

```cpp
#include <iostream>
#include <cmath>
using namespace std;

int main() {
    int n;
    cin >> n;
    for (int i = 1; i <= n; i++) {
        int s = (int)sqrt(i);       // 开根号并取整
        if (s * s == i) {            // 平方回去看看是不是等于 i
            cout << i << endl;
        }
    }
    return 0;
}
```

**方法二（换个角度枚举，更聪明）**：

> 老师启发：与其检查每个数是不是完全平方数，不如直接枚举"谁的平方在范围内"。

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    for (int i = 1; i * i <= n; i++) {
        cout << i * i << endl;
    }
    return 0;
}
```

> **引导提问**：方法一循环了 N 次，方法二循环了多少次？只有 √N 次！快了好多倍！这就是**缩小枚举范围**的威力。

### 例题 2：找出所有的水仙花数

**题目**：输出所有的三位水仙花数。水仙花数是指一个三位数，其各位数字的立方和等于该数本身。

**讲解思路**：
1. 枚举什么？——枚举所有三位数，即 100 到 999。
2. 条件是什么？——百位³ + 十位³ + 个位³ == 这个数本身。
3. 如何拆位？用 `/` 和 `%`：百位 = i/100，十位 = i/10%10，个位 = i%10。

```cpp
#include <iostream>
using namespace std;

int main() {
    for (int i = 100; i <= 999; i++) {
        int a = i / 100;        // 百位
        int b = i / 10 % 10;    // 十位
        int c = i % 10;         // 个位
        if (a*a*a + b*b*b + c*c*c == i) {
            cout << i << endl;
        }
    }
    return 0;
}
```

**运行结果**：

```
153
370
371
407
```

> **成就感设计**：让学生运行后数一数，一共只有 4 个水仙花数！电脑试了 900 个数，帮我们在一眨眼之间找出来了。

### 例题 3：百钱买百鸡（简化版）

**题目**：公鸡 5 元一只，母鸡 3 元一只，小鸡 1 元买 3 只。用 100 元钱买 100 只鸡，每种至少 1 只，有哪些买法？输出所有方案（公鸡数 母鸡数 小鸡数）。

**讲解思路**：
1. 枚举什么？——枚举公鸡数量 x 和母鸡数量 y。
2. 小鸡数量 z = 100 - x - y，不用枚举，直接算。
3. 条件是什么？——总价 = 5x + 3y + z/3 == 100，且 z > 0 且 z 能被 3 整除。
4. 枚举范围：x 从 1 到 19（至少 1 只，至多 100÷5=20，留余地给其他鸡），y 从 1 到 32。

```cpp
#include <iostream>
using namespace std;

int main() {
    int count = 0;  // 记录方案数
    for (int x = 1; x <= 19; x++) {           // 枚举公鸡数量
        for (int y = 1; y <= 32; y++) {        // 枚举母鸡数量
            int z = 100 - x - y;               // 小鸡数量直接算
            if (z >= 1 && z % 3 == 0) {        // 小鸡至少1只，且能3只3只地买
                int cost = 5 * x + 3 * y + z / 3;  // 算总花费
                if (cost == 100) {
                    cout << "公鸡:" << x << " 母鸡:" << y << " 小鸡:" << z << endl;
                    count++;
                }
            }
        }
    }
    cout << "共有 " << count << " 种买法" << endl;
    return 0;
}
```

**运行结果**：

```
公鸡:4 母鸡:18 小鸡:78
公鸡:8 母鸡:11 小鸡:81
公鸡:12 母鸡:4 小鸡:84
共有 3 种买法
```

> **教学重点回顾**：这道题我们只用了两重循环而不是三重循环，因为知道了公鸡和母鸡的数量，小鸡数量可以直接算出来。**能算就不枚举**——这就是优化！

## 七、常见错误与应对

### 错误 1：枚举范围写错

```cpp
// ❌ 要找三位数的水仙花数，却从 1 开始
for (int i = 1; i <= 999; i++)

// ✅ 三位数从 100 开始
for (int i = 100; i <= 999; i++)
```

> **应对**：做题时先在草稿纸上写清楚"答案最小是几，最大是几"，再写代码。

### 错误 2：拆位搞混

```cpp
// ❌ 十位写成了 i / 10（百位数没去掉）
int b = i / 10;     // 当 i=153 时，b=15，不对！

// ✅ 应该取模
int b = i / 10 % 10;  // 当 i=153 时，b=5，正确！
```

> **应对**：用具体数字代入验算。比如 i=153，手算一遍 `153/10=15`，`15%10=5`，没问题！

### 错误 3：百钱买百鸡中忘记检查小鸡能被 3 整除

```cpp
// ❌ 少了 z % 3 == 0 的检查
if (z >= 1) {
    int cost = 5 * x + 3 * y + z / 3;  // z不能被3整除时，z/3会丢失精度！
```

> **应对**：回到题意——"1 元买 3 只小鸡"，所以小鸡必须 3 只 3 只地买，数量一定是 3 的倍数。

### 错误 4：把 `==` 写成 `=`

```cpp
// ❌ 赋值而不是判断
if (s * s = i)

// ✅ 双等号才是判断相等
if (s * s == i)
```

> **应对**：老口诀——"一个等号是赋值，两个等号是比较"。看到 `if` 就检查一下等号。
