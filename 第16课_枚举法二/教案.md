### 教学重点
- 多重循环（二重、三重）枚举的写法与枚举范围确定

### 教学难点
- 从"暴力双重循环"到"优化为单重循环"的思维跳跃——学生容易觉得"能跑出来就行"，需要引导体会优化的价值

- 循环变量之间的**关联关系**——什么时候变量是独立的，什么时候一个能由另一个算出来

## 四、讲稿提纲

### 4.1 复习热身（5 分钟）

> 同学们，上节课我们学了一个超级厉害的方法叫"枚举法"。谁还记得枚举法是什么意思？
>
> 对了！就是**一个一个去试**，把所有可能都检查一遍，找到满足条件的答案。就像你丢了一块橡皮，你从书包第一层找到最后一层，每一层都翻一翻——这就是枚举！
>
> 上节课我们用的是**一个循环**，只需要试一个东西。但是今天，我们要升级了——如果要**同时试两个甚至三个东西**，怎么办呢？

**小游戏热身**：老师说一个数字 N（比如 10），学生快速说出两个加起来等于 N 的正整数组合（口头抢答）。引出"要找两个数，就需要两个循环"。

---

### 4.2 讲解1：从单重到双重——鸡兔同笼（15 分钟）

#### 故事引入

> 古时候有一个农夫，他养了一群鸡和兔子，关在同一个笼子里。他数了数，笼子里一共有 **35 个头**，**94 只脚**。请问：鸡有几只？兔有几只？
>
> 这道题，如果你是农夫，你会怎么猜？
>
> 最笨的办法：我先猜鸡有 0 只，那兔就有 35 只，脚有 35×4 = 140 只，太多了！再猜鸡有 1 只，兔有 34 只，脚有 1×2 + 34×4 = 138，还是太多……一直猜下去，直到脚数刚好 94。
>
> 等一下，这不就是**枚举法**吗？！

#### 方法一：双重循环暴力枚举

> 我们让计算机帮我们猜。用两个循环：
> - 外层循环：鸡从 0 到 35 一个一个试
> - 内层循环：兔从 0 到 35 一个一个试
> - 每次检查：头数对不对？脚数对不对？

**板书要点**：画一个表格，行是鸡的数量，列是兔的数量，打勾的是满足条件的。让学生直观看到"双重循环就是在一个大表格里一格一格检查"。

#### 方法二：优化——用一个变量推另一个

> 等等，同学们发现没有——如果我知道鸡有 `a` 只，那兔一定有 `35 - a` 只（因为总共 35 个头）！我根本不需要第二个循环去猜兔的数量！
>
> 这就像你知道全班 40 人，男生有 `a` 个，那女生一定是 `40 - a` 个。你不需要再一个一个数女生了。
>
> 这样，两个循环变成一个循环，计算机跑得更快了！

**关键比喻**：双重循环像是在一个大棋盘上每个格子都走一遍（35×35 = 1225 次）；优化后只走一行就够了（35 次）。就像走迷宫，聪明的人会抄近路！

---

### 4.3 讲解2：填运算符问题（15 分钟）

#### 问题引入

> 现在来一个更好玩的问题：
>
> `□ ○ □ ○ □ = 6`
>
> 三个方框里填 1、2、3（每个数只用一次），两个圆圈里填 + 或 -，使得等式成立。有哪些方案？
>
> 这个问题有两种东西要枚举：**数字的排列**和**运算符的选择**。我们今天先学运算符的枚举。

#### 运算符枚举的技巧

> 假如数字固定是 1、2、3，我们只枚举运算符。每个运算符有两种选择（+ 或 -），两个运算符就是 2×2 = 4 种组合。
>
> 我们可以用数字来代表运算符：0 代表加法，1 代表减法。然后用循环来枚举！

**思维引导**：从"觉得无从下手"到"用数字代替符号，转化成循环问题"——这是编程思维的核心：**把不熟悉的问题变成熟悉的问题**。

---

### 4.4 讲解3：三重循环——数字拆分（15 分钟）

#### 问题引入

> 把数字 10 拆成三个正整数的和，有多少种拆法？（注意：1+2+7 和 2+1+7 算**同一种**。）
>
> 这次我们需要枚举三个数 a、b、c，满足 a + b + c = 10。
>
> 如果不加任何限制，就要三重循环。但如果我们要求 a ≤ b ≤ c（从小到大排列），就可以大大减少枚举的范围！

#### 范围确定技巧

> a 从 1 开始，最大是多少？如果 a = b = c，那 3a = 10，a 最大是 3（因为 3×3 = 9 < 10，4×3 = 12 > 10）。
>
> b 从 a 开始（因为 b ≥ a），最大到哪里？b 最大时 b = c，所以 a + 2b ≤ 10，b ≤ (10 - a) / 2。
>
> c 不用枚举！c = 10 - a - b，直接算出来就好。

**生活比喻**：这就像分糖果。你要把 10 颗糖分给三个小朋友，第一个人拿的不能比第二个多，第二个不能比第三个多。你先让第一个人挑（最少的），再让第二个人挑，第三个人自动拿剩下的。

#### 总结：从暴力到巧妙

> 同学们，今天我们学了一个很重要的道理：
>
> 1. **先写出暴力方法**——能解决问题就是好方法
> 2. **再想想能不能优化**——少一层循环，快很多倍
> 3. **确定枚举范围**——范围越精确，程序越快
>
> 这就像考试做题：先用笨办法拿到分，再想巧办法拿更高的分！

## 五、课堂例题

### 例题1：鸡兔同笼（入门）

**题目**：笼子里有鸡和兔共 `n` 个头，`m` 只脚。输出鸡和兔各有多少只。如果无解输出 `No answer`。

**输入样例**：`35 94`
**输出样例**：`23 12`（23只鸡，12只兔）

#### 方法一：双重循环（暴力版）

**讲解思路**：
- 两个循环分别枚举鸡和兔的数量
- 检查两个条件：头数之和 = n，脚数之和 = m
- 这是最直观的"枚举法两步走"：先列举，再筛选

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    bool found = false;
    
    // 枚举鸡的数量
    for (int chicken = 0; chicken <= n; chicken++) {
        // 枚举兔的数量
        for (int rabbit = 0; rabbit <= n; rabbit++) {
            // 筛选：头数和脚数都要对
            if (chicken + rabbit == n && chicken * 2 + rabbit * 4 == m) {
                cout << chicken << " " << rabbit << endl;
                found = true;
            }
        }
    }
    
    if (!found) {
        cout << "No answer" << endl;
    }
    
    return 0;
}
```

#### 方法二：单重循环（优化版）

**讲解思路**：
- 关键发现：知道鸡的数量 `a`，兔的数量就是 `n - a`
- 只需要一个循环枚举鸡，兔由鸡推出
- 只需检查一个条件：脚数是否等于 m

```cpp
#include <iostream>
using namespace std;

int main() {
    int n, m;
    cin >> n >> m;
    bool found = false;
    
    for (int chicken = 0; chicken <= n; chicken++) {
        int rabbit = n - chicken;  // 用鸡推出兔
        if (chicken * 2 + rabbit * 4 == m) {
            cout << chicken << " " << rabbit << endl;
            found = true;
        }
    }
    
    if (!found) {
        cout << "No answer" << endl;
    }
    
    return 0;
}
```

**对比引导**：让学生分别运行两个版本，对比循环次数。暴力版循环 n×n 次，优化版只循环 n 次。当 n = 1000 时，暴力版要检查一百万次，优化版只要一千次——差了 1000 倍！

### 例题2：填运算符（进阶）

**题目**：在 `1 ○ 2 ○ 3 ○ 4` 的三个圆圈中填入 `+` 或 `-`，使得结果等于 4。输出所有方案。

**输入**：无
**输出示例**：
```
1+2-3+4=4
1-2+3+4=4  （虚拟示例，以实际计算为准）
```

**讲解思路**：
- 3 个运算符位置，每个位置 2 种选择，共 2×2×2 = 8 种组合
- 用 0 表示 `+`，用 1 表示 `-`
- 三重循环枚举三个运算符
- 计算结果，筛选等于 4 的方案

```cpp
#include <iostream>
using namespace std;

int main() {
    // op1, op2, op3 分别表示三个运算符
    // 0 表示 +，1 表示 -
    for (int op1 = 0; op1 <= 1; op1++) {
        for (int op2 = 0; op2 <= 1; op2++) {
            for (int op3 = 0; op3 <= 1; op3++) {
                // 计算结果
                int result = 1;  // 从第一个数开始
                
                if (op1 == 0) result = result + 2;
                else          result = result - 2;
                
                if (op2 == 0) result = result + 3;
                else          result = result - 3;
                
                if (op3 == 0) result = result + 4;
                else          result = result - 4;
                
                // 筛选
                if (result == 4) {
                    // 输出算式
                    cout << "1";
                    if (op1 == 0) cout << "+"; else cout << "-";
                    cout << "2";
                    if (op2 == 0) cout << "+"; else cout << "-";
                    cout << "3";
                    if (op3 == 0) cout << "+"; else cout << "-";
                    cout << "4";
                    cout << "=" << result << endl;
                }
            }
        }
    }
    
    return 0;
}
```

**运行结果**：
```
1+2-3+4=4
1-2+3+4=4 （虚拟示例，以实际运行为准）
```

**教学小贴士**：让学生先手动列出 8 种组合，在纸上算出结果，再和程序输出对比，增强对"计算机帮我们一个个试"的体感。

### 例题3：数字拆分（提高）

**题目**：输入正整数 `n`（n ≤ 20），把 `n` 拆分成三个正整数之和（要求 a ≤ b ≤ c），输出所有拆分方案和方案总数。

**输入样例**：`7`
**输出样例**：
```
1 1 5
1 2 4
1 3 3
2 2 3
4
```

**讲解思路**：
- 要求 a ≤ b ≤ c，避免重复
- a 的范围：1 到 n/3（因为 a 是最小的，三个 a 不超过 n）
- b 的范围：a 到 (n-a)/2（因为 b 是中间的，b 和 c 分剩下的，b ≤ c）
- c 不用枚举，直接 c = n - a - b
- 最后检查 c ≥ b（其实由 b 的范围已经保证了）

```cpp
#include <iostream>
using namespace std;

int main() {
    int n;
    cin >> n;
    int count = 0;
    
    for (int a = 1; a <= n / 3; a++) {               // a 是最小的
        for (int b = a; b <= (n - a) / 2; b++) {      // b 从 a 开始，不超过剩余的一半
            int c = n - a - b;                         // c 直接算出来
            cout << a << " " << b << " " << c << endl;
            count++;
        }
    }
    
    cout << count << endl;
    
    return 0;
}
```

**优化讲解**：
> 如果不加 a ≤ b ≤ c 的限制，三重循环要跑 n×n×n 次。加了限制后，循环次数大大减少。而且 c 不用循环，直接算出来，又省了一层！
>
> 这就是枚举优化的秘诀：**能算出来的就不要去试，能缩小范围的就尽量缩小。**

## 七、常见错误与应对

### 错误1：枚举范围搞错，漏掉答案

```cpp
// ❌ 错误：鸡从 1 开始，漏掉了鸡为 0 的情况
for (int chicken = 1; chicken <= n; chicken++)

// ✅ 正确：鸡可以为 0（万一全是兔呢！）
for (int chicken = 0; chicken <= n; chicken++)
```

**应对**：提醒学生想清楚"最小值是什么？最大值是什么？"，边界值一定要试。可以让学生**手动验证**第一个和最后一个值是否合理。

---

### 错误2：优化时忘记检查推导出的变量是否合法

```cpp
// ❌ 错误：兔的数量可能为负数！
for (int chicken = 0; chicken <= 100; chicken++) {
    int rabbit = n - chicken;  // 如果 chicken > n，rabbit 就是负数了
    // 没有检查 rabbit >= 0
}

// ✅ 正确：要检查推导出的变量是否合理
for (int chicken = 0; chicken <= n; chicken++) {
    int rabbit = n - chicken;
    if (rabbit >= 0 && chicken * 2 + rabbit * 4 == m) {
        cout << chicken << " " << rabbit << endl;
    }
}
```

**应对**：告诉学生一个口诀："**推出来的数，要检查有没有越界！**"

---

### 错误3：三重循环写成三个并列循环

```cpp
// ❌ 错误：三个循环是并列的，不是嵌套的！
for (int a = 1; a <= n; a++) { ... }
for (int b = 1; b <= n; b++) { ... }
for (int c = 1; c <= n; c++) { ... }

// ✅ 正确：三个循环要嵌套
for (int a = 1; a <= n; a++) {
    for (int b = a; b <= n; b++) {
        int c = n - a - b;
        ...
    }
}
```

**应对**：用"抽屉比喻"——第一层循环是打开第一个抽屉，对于第一个抽屉里的每样东西，都要打开第二个抽屉看一遍；对于第二个抽屉里的每样东西，又要打开第三个抽屉看一遍。**一层套一层，不是排排站！**

---

### 错误4：数字拆分题没有去重

```cpp
// ❌ 错误：1+2+4 和 2+1+4 和 4+1+2 都会被输出，重复了
for (int a = 1; a < n; a++)
    for (int b = 1; b < n; b++)
        for (int c = 1; c < n; c++)

// ✅ 正确：要求 a <= b <= c，避免重复
for (int a = 1; a <= n/3; a++)
    for (int b = a; b <= (n-a)/2; b++)
```

**应对**：在黑板上写出 1+2+4、2+1+4、4+2+1，问学生"这三个拆法本质一样吗？"引导学生理解**排列和组合的区别**，从而接受 a ≤ b ≤ c 的约束。

---

### 错误5：忘记计数变量初始化或位置放错

```cpp
// ❌ 错误：count 放在循环里面，每次被重置为 0
for (int a = 0; a <= n; a++) {
    int count = 0;  // 每次循环都归零了！
    ...
    count++;
}

// ✅ 正确：count 在循环外面定义
int count = 0;
for (int a = 0; a <= n; a++) {
    ...
    count++;
}
```

**应对**：提醒学生——计数器就像体育课报数的同学，他要**站在队伍外面**数，不能站在队伍里面，否则每次都从 1 开始报了。

## 九、下节课预告

> 同学们，今天我们学会了用**多重循环**来枚举，还学会了怎么让枚举变得更**聪明**、更**快**。
>
> 下节课，我们要学一种新的方法，叫做**"模拟法"**。什么是模拟呢？就是题目让你做什么，你就让计算机**一步一步照着做**——就像玩角色扮演游戏一样，计算机扮演题目里的角色，按照规则一步步执行。
>
> 下节课我们会模拟一些好玩的场景，比如"报数游戏"和"机器人走路"。很有意思哦，敬请期待！
